# Phase 12.1 Plan 07: Admin Table Pagination, Sort Affordance, and Settings Reset Summary

Admin content table now has full pagination (25 rows/page), visual sort affordance on inactive columns, icon column headers; SettingsScreen now shows an amber "Unsaved changes" indicator and a Reset button to discard form edits.

## What Was Built

### Task 1 - Admin Table UX

- **Inactive sort affordance:** `SortIcon` now renders `ChevronsUpDown` at `opacity-40` for unsorted columns (was `null`) - users can now see all columns are sortable.
- **Icon column headers:** Locale column uses `Globe` icon + sr-only label; Actions column uses `Settings2` icon - compact headers for narrow columns without losing accessibility.
- **Pagination:** `PAGE_SIZE = 25`, `page` state, `paginated` useMemo slice, `totalPages` derived value. `useEffect` resets page to 1 whenever filter or sort changes. Prev/Next controls with "Page X of Y" info text rendered below the table only when `totalPages > 1`.
- **i18n:** Added `paginationPrev`, `paginationNext`, `paginationInfo` keys to both locale files under `admin.content`.

### Task 2 - SettingsScreen

- **Unsaved indicator:** Amber `"Modifiche non salvate"` / `"Unsaved changes"` span appears inline with the Save button group when `isDirty`.
- **Reset button:** `"Annulla modifiche"` / `"Discard changes"` button - calls `handleReset` (useCallback) which restores all 5 dirty-tracked fields (`firstName`, `lastName`, `voiceGender`, `voiceAutoListen`, `defaultPersonaId`) to the saved `user` values. Only shown when `isDirty`.
- **i18n:** Added `settings.unsavedChanges` and `settings.reset` keys to both locale files.
