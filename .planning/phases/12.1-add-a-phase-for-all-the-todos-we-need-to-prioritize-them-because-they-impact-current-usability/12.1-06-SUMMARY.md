# Phase 12.1 Plan 06: Profile i18n, Hash Nav, and Merchant Privacy Check Summary

**22 category slugs localized in Italian and English, estimated-from-transactions source label added, hash-nav IDs on profile sections with scroll-to-anchor on mount, and amber privacy notice for likely-private-individual counterparties in manual categorization**

## Performance

- **Duration:** 5 min
- **Completed:** 2026-04-02
- **Tasks:** 2
- **Files modified:** 4

## Accomplishments

- Added `profile.categories.*` i18n entries for all 22 category slugs to `it.json` and `en.json` - category chart and option pickers now show "Supermercato", "Ristoranti", etc. instead of English slugs
- Added `profile.sourceEstimated` key (`"stimato dalle transazioni"` / `"estimated from transactions"`) and wired it to `DATA_SOURCE_LABELS` in `ProfileScreen.tsx`
- Refactored `getCategoryChartData` to accept a `categoryLabel` callback - enables locale-aware labels without coupling the pure function to React hooks
- Added `profile.privateIndividualNotice` key to both locales
- Added `looksLikePrivateIndividual()` heuristic to `UncategorizedScreen.tsx` - checks for 2+ title-cased words with no business suffixes
- Privacy notice (amber text) shown below counterpart name when heuristic returns true
- Category `<option>` labels now use `t('profile.categories.{cat}', { defaultValue: ... })`
- Added `id` attributes to five profile section elements: `income`, `spending`, `income-vs-expenses`, `insights`, `confirm`
- Added `useEffect` on mount to `scrollIntoView` if `window.location.hash` matches a section ID

## Task Commits

1. **Task 1: i18n keys + category localization + merchant privacy notice** - `b6390c6` (feat)
2. **Task 2: Hash navigation IDs + scroll-to-anchor on mount** - `0417413` (feat)

## Files Created/Modified

- `senso/src/i18n/locales/it.json` - Added `profile.categories.*` (22 entries), `profile.sourceEstimated`, `profile.privateIndividualNotice`
- `senso/src/i18n/locales/en.json` - Mirrored all new keys in English
- `senso/src/features/profile/ProfileScreen.tsx` - `getCategoryChartData` refactor, `DATA_SOURCE_LABELS` extended, hash nav `useEffect`, section `id` attributes
- `senso/src/features/profile/UncategorizedScreen.tsx` - `looksLikePrivateIndividual` heuristic, amber notice in JSX, localized `<option>` labels

## Decisions Made

- `getCategoryChartData` gets `categoryLabel` as a parameter instead of being moved inside the component - keeps the pure function testable and outside render, locale lookup stays in the component call site
- `looksLikePrivateIndividual` requires ≥2 words (not just 1) - "Amazon" is a single word that passes title-case but should not trigger the notice
- Hash nav uses native `scrollIntoView` on mount rather than any router integration - the hash is set before React renders; a one-time `useEffect([])` is sufficient and zero-dependency

None - plan executed exactly as written.
