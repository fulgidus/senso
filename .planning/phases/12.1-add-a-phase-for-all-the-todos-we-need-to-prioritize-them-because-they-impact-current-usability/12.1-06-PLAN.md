Find the uncategorized transactions screen. Check:


The manual categorization flow currently just shows a category picker. For privacy compliance, when a user categorizes a transaction:
1. Check if the counterparty name looks like a private individual (a person's name) rather than a business.
2. If it looks like a person, show a notice: "Questo pagamento sembra andare a una persona privata. Per la tua privacy, considera di non categorizzarlo con dettagli personali."

**Implementation:**
- Add a simple heuristic function `looksLikePrivateIndividual(name: string): boolean`:
  - Returns `true` if the name matches common Italian name patterns (starts with common first names, contains typical surname patterns, etc.)
  - Simple approach: check if the name contains only words that are all Title Case (no company suffixes like s.r.l., s.p.a., sas, snc, bank, etc.)
  
```typescript
function looksLikePrivateIndividual(name: string): boolean {
  if (!name) return false
  const businessIndicators = /\b(srl|spa|sas|snc|snc|banca|bank|poste|amazon|google|apple|netflix|spotify|telecom|vodafone|tim|enel|eni|fineco|unicredit|intesa|sanpaolo|bnl|bnp|paypal|stripe|klarna)\b/i
  if (businessIndicators.test(name)) return false
  // If all words are title-cased and no business indicators, likely a person
  const words = name.trim().split(/\s+/)
  return words.every(w => /^[A-ZÁÀÈÉÌÍÒÓÙÚ][a-záàèéìíòóùú]+$/.test(w))
}
```

- In the uncategorized transaction component, when rendering a transaction for categorization, if `looksLikePrivateIndividual(transaction.description)` returns true, show a small amber notice below the transaction name with the i18n key `profile.privateIndividualNotice`.
