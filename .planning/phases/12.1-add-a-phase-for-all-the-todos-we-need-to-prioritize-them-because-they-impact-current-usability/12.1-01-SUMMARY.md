# Phase 12.1 Plan 01: Token Renewal and Expired-Token Redirect Summary

**401-intercept with token refresh retry wired into api-client.ts and useAuth hook via callback injection pattern to prevent circular imports**

## Accomplishments
- `apiRequest` now intercepts 401 responses, calls `onUnauthorized()` to refresh the token, and retries the original request once with the new token
- `makeOnUnauthorized` factory added to `session.ts`: reads stored refresh token, calls internal `refresh()`, returns new access token or `null` (and navigates to `/auth`) on double expiry
- `useAuth` hook now exports `onUnauthorized` callback (built via `useMemo` + `useNavigate`) so all feature screens can pass it to API calls for transparent session renewal

## Task Commits

Each task was committed atomically:

1. **Task 1: Add 401-intercept with token refresh and retry to api-client.ts** - `4144b6a` (feat)

**Plan metadata:** committed in docs commit below

## Files Created/Modified
- `senso/src/lib/api-client.ts` - Added `onUnauthorized` callback to `RequestOptions`, `_isRetry` internal flag, 401-intercept retry logic
- `senso/src/features/auth/session.ts` - Added `makeOnUnauthorized` factory export; `refresh()` kept without `onUnauthorized` to prevent infinite loops
- `senso/src/features/auth/useAuth.ts` - Imports `makeOnUnauthorized` + `useNavigate`, creates `onUnauthorized` via `useMemo`, exports it in hook return

## Decisions Made
- **Callback injection pattern**: `api-client.ts` cannot import `session.ts` because `session.ts` imports `api-client.ts`. The `onUnauthorized` callback parameter keeps the coupling correct - callers inject the refresh logic.
- **No `onUnauthorized` on `refresh()` call**: The internal `refresh()` in `session.ts` deliberately omits `onUnauthorized` to prevent infinite retry loops when the refresh token itself is expired (double-expiry handled by the factory's catch block).
- **Factory with optional navigate**: `makeOnUnauthorized` accepts an optional `navigate` function so it can work both inside React hooks (with `useNavigate`) and in non-hook contexts if needed.
