# Phase 12.1 Plan 04: Error Boundaries, Pull-to-Refresh Fix, and Debug Nuke Token Summary

**React ErrorBoundary class component wrapping profile/chat routes, pull-to-refresh touchAction CSS fix on ProfileScreen and ChatScreen, and DebugScreen wired to onUnauthorized for reliable token refresh**

## Accomplishments
- Created reusable `ErrorBoundary` class component with recovery UI and reset capability - wraps ProfileScreen, UncategorizedScreen, and ChatRoutes
- Fixed pull-to-refresh gesture on ProfileScreen by adding `style={{ touchAction: "pan-x" }}` - enables `preventDefault` in `touchmove` listener despite browser passive scroll optimization
- Applied same touchAction fix to ChatScreen message list div (shares same `usePullToRefresh` pattern)
- DebugScreen nuke/restart/purge buttons now pass `onUnauthorized` from `useAuth()` to `apiRequest` - expired tokens auto-refresh instead of silently failing

- `senso/src/features/debug/DebugScreen.tsx` - Added `useAuth` import, `onUnauthorized` passed to all `apiRequest` calls

## Decisions Made
- Hardcoded Italian error strings in ErrorBoundary fallback (not i18n keys) - the i18n system itself could be the cause of the crash; fallback must be self-contained
- Used `touchAction: "pan-x"` not `"none"` - pan-x allows the browser to still handle horizontal swipes natively, only blocking the vertical default to enable our custom pull-down gesture
- Used `useAuth().onUnauthorized` in DebugScreen rather than directly importing `makeOnUnauthorized` - consistent with the rest of the codebase pattern
