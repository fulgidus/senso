<task type="auto">
  <name>Task 1: Add 401-intercept with token refresh and retry to api-client.ts</name>
  <files>senso/src/lib/api-client.ts</files>
  <action>
Upgrade `apiRequest` to handle token expiry automatically.

Current state: `apiRequest` throws `ApiClientError` with status 401 whenever the server returns 401. There is no retry logic. This means any request made after the 15-min access token TTL silently fails.

Implementation:

1. Change the `apiRequest` signature to accept an optional `onUnauthorized?: () => Promise<string | null>` callback. This callback is responsible for refreshing the token and returning the new access token (or `null` if refresh failed).

2. When `response.ok === false && response.status === 401`:
   - If `onUnauthorized` is provided AND this is not already a retry (add a `isRetry?: boolean` internal parameter), call `onUnauthorized()`.
   - If it returns a new token string, retry the request once with the new token in the Authorization header.
   - If it returns `null` (refresh also failed), re-throw `ApiClientError` with status 401.
   - If `onUnauthorized` is not provided, re-throw as before (backward compatible).

3. IMPORTANT: Do NOT import `session.ts` directly into `api-client.ts` - that would create a circular dependency (session.ts imports api-client.ts). The `onUnauthorized` callback pattern keeps the coupling correct.

4. Keep the function signature backward compatible: existing callers that don't pass `onUnauthorized` behave identically to today.

Example final interface:
```typescript
export async function apiRequest<T>(
  baseUrl: string,
  path: string,
  options: RequestOptions & {
    onUnauthorized?: () => Promise<string | null>
    _isRetry?: boolean
  } = {},
): Promise<T>
```

  <name>Task 2: Wire refresh-and-redirect into all authenticated API callers</name>
  <files>senso/src/features/auth/session.ts, senso/src/features/auth/useAuth.ts</files>
  <action>
Create a reusable `makeOnUnauthorized` factory in `session.ts` and wire it into the auth hook so all API calls in the app auto-refresh.

**In `session.ts`:**

Add and export:
```typescript
export function makeOnUnauthorized(navigate?: (to: string) => void): () => Promise<string | null> {
  return async () => {
    const refreshToken = readRefreshToken()
    if (!refreshToken) {
      clearTokens()
      navigate?.("/auth")
      return null
    }
    try {
      const payload = await refresh(refreshToken)  // already exported from session.ts
      return payload.accessToken
    } catch {
      clearTokens()
      navigate?.("/auth")
      return null
    }
  }
}
```

Note: `refresh()` is already defined in `session.ts` at line 182 and calls `apiRequest` WITHOUT an `onUnauthorized` callback (to avoid infinite retry loops). This is correct - do not add `onUnauthorized` to the refresh call itself.

**In `useAuth.ts`:**

1. Import `makeOnUnauthorized` from `session.ts`.
2. Import `useNavigate` from `react-router-dom`.
3. Inside the hook, create the callback:
   ```typescript
   const navigate = useNavigate()
   const onUnauthorized = useMemo(
     () => makeOnUnauthorized((to) => navigate(to)),
     [navigate],
   )
   ```
4. Export `onUnauthorized` from the hook's return value so feature screens can pass it to API calls.

**IMPORTANT re #15 (nuke button):** The `DebugScreen.tsx` uses `readAccessToken() ?? ""` directly. After this fix, if the access token is expired, `apiRequest` will auto-refresh when `onUnauthorized` is wired. For now, `DebugScreen` doesn't use the hook's `onUnauthorized` - that will be picked up in Plan 04. This plan only establishes the mechanism.

**Do NOT** change `SettingsScreen`, `DebugScreen`, or other feature files - those are separate plans.
