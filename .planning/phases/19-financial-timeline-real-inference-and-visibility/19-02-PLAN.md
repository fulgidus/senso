# Plan 19-02 - Non-Ledger Timeline Triggers

## Objective

Wire payslip and utility bill ingestion to trigger timeline events automatically,
so the timeline populates without requiring transaction volume thresholds.

---

<task id="19-02-01" title="Payslip → income_shift trigger">
  <objective>
    After `ProfileService.enrich_from_payslip()` updates `verified_income_sources`,
    compare the new `net_salary` to the previous entry. If difference >5%, create or
    update an `income_shift` timeline event.
  </objective>
  <read_first>
    - api/app/services/profile_service.py - enrich_from_payslip()
    - api/app/ingestion/adaptive.py - _run_timeline_inference(), upsert_timeline_event()
  </read_first>
  <action>
    In `enrich_from_payslip()`, after appending to `verified_income_sources`:
    ```python
    if len(sources) >= 2:
        prev = sources[-2]["net_salary"]
        curr = sources[-1]["net_salary"]
        if abs(curr - prev) / prev > 0.05:
            direction = "increase" if curr > prev else "decrease"
            upsert_timeline_event(user_id, db, "income_shift", event_date=pay_period_date,
                payload={"amount": curr, "change_pct": round((curr-prev)/prev*100,1),
                         "direction": direction, "source": "payslip"})
    ```
  </action>
  <acceptance_criteria>
    - Payslip with salary 10% higher than previous → `income_shift` event
    - Same payslip re-ingested (blocked by dedup from 18-01) → no duplicate event
    - Pytest: `test_timeline_inference.py::test_payslip_triggers_income_shift`
  </acceptance_criteria>
</task>

<task id="19-02-02" title="New utility bill → subscription_accumulation trigger">
  <objective>
    After `ProfileService.enrich_from_utility_bill()` adds to `fixed_expenses`,
    if total fixed expenses increased by >10% or a new provider was added,
    create a `subscription_accumulation` event.
  </objective>
  <read_first>
    - api/app/services/profile_service.py - enrich_from_utility_bill()
  </read_first>
  <action>
    In `enrich_from_utility_bill()`, after updating `fixed_expenses`:
    ```python
    total_before = sum(e["monthly_amount"] for e in expenses_before)
    total_after = sum(e["monthly_amount"] for e in profile.fixed_expenses)
    if total_after > total_before * 1.10 or is_new_provider:
        upsert_timeline_event(user_id, db, "subscription_accumulation",
            event_date=date.today(),
            payload={"total_monthly": total_after, "new_provider": provider})
    ```
  </action>
  <acceptance_criteria>
    - Adding a new €50/month utility → `subscription_accumulation` event
    - Updating same provider's amount (not new) → no event if <10% change
    - Pytest: `test_timeline_inference.py::test_utility_triggers_subscription`
  </acceptance_criteria>
</task>
