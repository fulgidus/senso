# Plan 19-04 - Timeline Tab Visual Improvements

## Objective

Redesign the TimelineTab to show a visual chronological timeline with relative date
labels, transaction counts, an empty state CTA, and a notification badge on the profile
tab when new events are present.

---

<task id="19-04-01" title="Visual timeline with date axis">
  <objective>
    Replace the flat card list with a chronological layout: a vertical connector line
    with event cards branching off it. Each card shows relative date ("3 mesi fa"),
    event icon, summary, and a "basato su N transazioni" footnote.
  </objective>
  <read_first>
    - senso/src/features/profile/TimelineTab.tsx - current implementation
    - senso/src/locales/it.json - existing timeline.* keys
  </read_first>
  <action>
    Layout:
    ```tsx
    <ol className="relative border-l border-border ml-4 space-y-6">
      {events.map(event => (
        <li key={event.id} className="ml-6">
          {/* dot on the line */}
          <span className="absolute -left-3 flex h-6 w-6 items-center justify-center 
                           rounded-full bg-primary/10 ring-2 ring-background">
            <EventIcon className="h-3 w-3 text-primary" />
          </span>
          {/* card */}
          <div className="rounded-lg border bg-card p-4">
            <div className="flex items-center justify-between">
              <h3 className="font-medium">{t(`timeline.${event.event_type}`)}</h3>
              <time className="text-xs text-muted-foreground">
                {formatRelativeDate(event.event_date, locale)}
              </time>
            </div>
            {event.payload?.transaction_count && (
              <p className="text-xs text-muted-foreground mt-1">
                {t("timeline.basedOnTransactions", {count: event.payload.transaction_count})}
              </p>
            )}
            {/* dismiss / context actions unchanged */}
          </div>
        </li>
      ))}
    </ol>
    ```
    Add `formatRelativeDate(isoDate, locale)` utility using `Intl.RelativeTimeFormat`.
  </action>
  <acceptance_criteria>
    - Events render in chronological order (newest first)
    - Date shows relative format: "2 giorni fa", "3 mesi fa"
    - Visual connector line visible between events
    - `pnpm build` clean
  </acceptance_criteria>
</task>

<task id="19-04-02" title="Empty state CTA + profile tab notification badge">
  <objective>
    Show a helpful empty state when no timeline events exist (with CTA to upload
    documents). Add a red badge to the Timeline tab button in ProfileScreen when
    undismissed events are present.
  </objective>
  <read_first>
    - senso/src/features/profile/TimelineTab.tsx - current empty handling
    - senso/src/features/profile/ProfileScreen.tsx - tab bar rendering
    - senso/src/locales/it.json - add keys
  </read_first>
  <action>
    Empty state:
    ```tsx
    {events.length === 0 && !loading && (
      <div className="text-center py-12 text-muted-foreground">
        <Clock className="mx-auto h-10 w-10 mb-3 opacity-30" />
        <p className="font-medium">{t("timeline.emptyTitle")}</p>
        <p className="text-sm mt-1">{t("timeline.emptyHint")}</p>
        <Button variant="outline" className="mt-4" onClick={() => navigate("/upload")}>
          {t("timeline.emptyCta")}
        </Button>
      </div>
    )}
    ```

    In ProfileScreen, pass `hasNewEvents` prop:
    ```tsx
    <button onClick={() => setActiveTab("timeline")} className="relative ...">
      {t("timeline.tabLabel")}
      {hasNewEvents && (
        <span className="absolute -top-1 -right-1 h-2 w-2 rounded-full bg-destructive" />
      )}
    </button>
    ```
    `hasNewEvents` = `events.some(e => !e.dismissed && isRecent(e.created_at))`.

    Add i18n keys: `timeline.emptyTitle`, `timeline.emptyHint`, `timeline.emptyCta`,
    `timeline.basedOnTransactions` to both `it.json` and `en.json`.
  </action>
  <acceptance_criteria>
    - Empty state renders with CTA button when no events
    - Red dot appears on Timeline tab when undismissed recent events exist
    - Dot disappears after all events are dismissed
    - i18n keys present in both locale files
  </acceptance_criteria>
</task>
