# Plan 19-03 — Coach System Prompt + get_timeline_events Tool

## Objective

Inject a condensed timeline summary into the coach system prompt and add a
`get_timeline_events` LLM tool so the coach can query specific event types on demand.

---

<task id="19-03-01" title="timeline_block.j2 template + prompt injection">
  <objective>
    Create a new Jinja2 template `api/app/coaching/prompts/timeline_block.j2` that
    renders a condensed summary of the user's most recent (non-dismissed) timeline
    events. Inject this block into the system prompt when events exist.
  </objective>
  <read_first>
    - api/app/coaching/prompts/context_block.j2 — existing profile block pattern
    - api/app/coaching/service.py — _build_system_prompt(), profile context injection
    - api/app/db/models.py — FinancialTimeline
  </read_first>
  <action>
    `timeline_block.j2`:
    ```jinja2
    {% if timeline_events %}
    ## Eventi finanziari recenti dell'utente
    {% for event in timeline_events[:3] %}
    - **{{ event.event_type | replace("_", " ") | title }}** ({{ event.event_date | dateformat }}):
      {{ event.summary or event.event_type }}
    {% endfor %}
    {% endif %}
    ```

    In `CoachingService._fetch_profile_context()`:
    - Query `FinancialTimeline` for non-dismissed events in last 180 days, limit 3
    - Pass `timeline_events` to template context
  </action>
  <acceptance_criteria>
    - System prompt contains timeline block when events exist (assert substring in pytest)
    - System prompt omits block when no events (no empty section)
    - `test_coaching_service.py::test_timeline_in_system_prompt` passes
  </acceptance_criteria>
</task>

---

<task id="19-03-02" title="get_timeline_events LLM tool">
  <objective>
    Register a `get_timeline_events` tool in the coaching tool suite alongside the
    existing `search_content` tool. The LLM can call it to fetch events by type.
  </objective>
  <read_first>
    - api/app/coaching/service.py — _SEARCH_CONTENT_TOOL schema, _tool_executor()
  </read_first>
  <action>
    ```python
    _GET_TIMELINE_TOOL = {
        "type": "function",
        "function": {
            "name": "get_timeline_events",
            "description": "Fetch the user's financial life events (income changes, moves, debt changes, etc.)",
            "parameters": {
                "type": "object",
                "properties": {
                    "types": {
                        "type": "array",
                        "items": {"type": "string",
                                  "enum": ["income_shift","major_purchase","subscription_accumulation",
                                           "extraordinary_income","relocation","debt_change"]},
                        "description": "Event types to fetch. Empty = all types."
                    }
                },
                "required": [],
                "additionalProperties": False
            }
        }
    }
    ```
    In `_tool_executor()`:
    ```python
    if name == "get_timeline_events":
        types = args.get("types", [])
        # query DB, return list of {event_type, event_date, summary, payload}
    ```
    Pass `[_SEARCH_CONTENT_TOOL, _GET_TIMELINE_TOOL]` to `complete_with_tools()`.
  </action>
  <acceptance_criteria>
    - Tool registered and callable in `_tool_executor`
    - Calling tool with `types=["income_shift"]` returns only income_shift events
    - `test_coaching_service.py::test_timeline_tool_callable` passes
  </acceptance_criteria>
</task>
