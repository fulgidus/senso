Append a "Voice Stack" section to `.planning/STATE.md` summarizing the working architecture so future phases don't re-investigate:

```markdown
## Voice Stack (Phase 24)

**STT (Speech-to-Text):**
- Primary: Web Speech API (Chrome/Edge/Safari) — real-time interim transcripts, zero latency
- Fallback: MediaRecorder → POST /coaching/stt → transcription provider
  - Provider selected via `STT_PROVIDER` env var (default: `"elevenlabs"`)
  - `elevenlabs`: ElevenLabs Scribe v1 — requires `ELEVENLABS_API_KEY`
  - `openai`: OpenAI Whisper — requires OpenAI API key via LLMConfig
- Hook: `useVoiceInput` (auto-detects backend at mount)
- Frontend availability: `isAvailable = true` when EITHER Web Speech API OR MediaRecorder present

**TTS (Text-to-Speech):**
- Primary: ElevenLabs via POST /coaching/tts → ObjectURL → HTMLAudioElement
  - Voice ID resolved from `api/app/personas/config.json` by persona + locale + gender
  - MinIO caching keyed by SHA-256(voice_id:text)
- Fallback: `window.speechSynthesis` (browser TTS) — activated when ElevenLabs returns 503
- Hook: `useTTS` (manages ObjectURL lifecycle, autoplay blocking, fallback badge)
- Orchestration: `useVoiceMode` (hold-to-talk, STT→TTS feedback loop prevention, auto-listen)

**Key config:**
- `ELEVENLABS_API_KEY` — required for both TTS and ElevenLabs STT
- `STT_PROVIDER` — selects STT backend (`elevenlabs` | `openai`)
- `voice_gender` in user profile — selects ElevenLabs voice variant
```

Do NOT overwrite existing STATE.md content. Append after the last section.
