# Plan 24-01: Fix TTS backend unit tests

## Objective

Fix 10 failing tests in `api/tests/test_tts.py`. Two distinct root causes:

1. **elevenlabs mock broken** (2 tests): `patch.dict("sys.modules", {"elevenlabs": MagicMock()})` does not stub `elevenlabs.types`, so `from elevenlabs.types import VoiceSettings` inside `tts.py` throws `ModuleNotFoundError: No module named 'elevenlabs.types'`.
2. **Stale voice ID constants** (8 tests): Hardcoded ElevenLabs voice IDs in the test assertions no longer match the values in `api/app/personas/config.json`.

## Verification Baseline

```bash
docker compose run --rm api uv run pytest tests/test_tts.py -q
# Current: 10 failed, 22 passed
```

## Task 1: Fix elevenlabs module mock in TTSService unit tests

<read_first>
- `api/tests/test_tts.py` — exact test code for `test_tts_service_returns_bytes_when_key_set` and `test_tts_service_truncates_long_text`
- `api/app/coaching/tts.py` — lines 120-148 (the `try` block with lazy `from elevenlabs import ElevenLabs` and `from elevenlabs.types import VoiceSettings`)
</read_first>

<action>
The two tests that mock the elevenlabs SDK use:

```python
with patch.dict("sys.modules", {"elevenlabs": MagicMock()}):
```

This only stubs `sys.modules["elevenlabs"]`. The `tts.py` code also does `from elevenlabs.types import VoiceSettings`, which needs `sys.modules["elevenlabs.types"]` to exist.

**Fix**: Add `"elevenlabs.types"` to the patch dict for BOTH tests. The `MagicMock()` for `elevenlabs.types` automatically makes `VoiceSettings` available as an attribute.

Replace in `test_tts_service_returns_bytes_when_key_set`:
```python
# BEFORE
with patch.dict("sys.modules", {"elevenlabs": MagicMock()}):
    import sys
    mock_elevenlabs_module = sys.modules["elevenlabs"]
    mock_elevenlabs_module.ElevenLabs.return_value = mock_client

    svc = TTSService(api_key="test-key")
    result = svc.speak("Hello world", "voice123", "it")
```

With:
```python
# AFTER
el_mock = MagicMock()
el_mock.ElevenLabs.return_value = mock_client
with patch.dict("sys.modules", {"elevenlabs": el_mock, "elevenlabs.types": MagicMock()}):
    svc = TTSService(api_key="test-key")
    result = svc.speak("Hello world", "voice123", "it")
```

Apply the same pattern to `test_tts_service_truncates_long_text`:

Replace:
```python
with patch.dict("sys.modules", {"elevenlabs": MagicMock()}):
    import sys
    mock_elevenlabs_module = sys.modules["elevenlabs"]
    mock_client = MagicMock()
    mock_client.text_to_speech.convert.return_value = b"audio"
    mock_elevenlabs_module.ElevenLabs.return_value = mock_client

    svc = TTSService(api_key="test-key")
    long_text = "x" * 3000
    svc.speak(long_text, "voice123", "it")

    # Check that the text passed to ElevenLabs is ≤ 2500 chars
    call_kwargs = mock_client.text_to_speech.convert.call_args.kwargs
    sent_text = call_kwargs.get("text", "")
    assert len(sent_text) <= 2500
```

With:
```python
el_mock = MagicMock()
mock_client = MagicMock()
mock_client.text_to_speech.convert.return_value = b"audio"
el_mock.ElevenLabs.return_value = mock_client
with patch.dict("sys.modules", {"elevenlabs": el_mock, "elevenlabs.types": MagicMock()}):
    svc = TTSService(api_key="test-key")
    long_text = "x" * 3000
    svc.speak(long_text, "voice123", "it")

    # Check that the text passed to ElevenLabs is ≤ 2500 chars
    call_kwargs = mock_client.text_to_speech.convert.call_args.kwargs
    sent_text = call_kwargs.get("text", "")
    assert len(sent_text) <= 2500
```

Also apply the same fix to `test_tts_service_raises_on_elevenlabs_failure` (already passing, but it uses the same pattern — hardening it prevents future breakage):
```python
el_mock = MagicMock()
mock_client = MagicMock()
mock_client.text_to_speech.convert.side_effect = RuntimeError("API error")
el_mock.ElevenLabs.return_value = mock_client
with patch.dict("sys.modules", {"elevenlabs": el_mock, "elevenlabs.types": MagicMock()}):
    svc = TTSService(api_key="test-key")
    with pytest.raises(TTSUnavailableError, match="ElevenLabs call failed"):
        svc.speak("Hello world", "voice123", "it")
```
</action>

<acceptance_criteria>
- `api/tests/test_tts.py` contains `"elevenlabs.types": MagicMock()` in `patch.dict` calls for `test_tts_service_returns_bytes_when_key_set` and `test_tts_service_truncates_long_text`
- `docker compose run --rm api uv run pytest tests/test_tts.py::test_tts_service_returns_bytes_when_key_set tests/test_tts.py::test_tts_service_truncates_long_text -v` exits 0
- No `import sys` left dangling inside the `with patch.dict` blocks for those two tests
