# Plan 36-02: CoachingService — Inject BehaviorLayer + FilterPipeline + EnrichmentCaps

## Goal

Remove hardcoded `PERSONAS_DIR` paths, hardcoded `settings.coaching_cap_*` usage, and direct `SafetyScanner()` instantiation from `CoachingService`. Inject `BehaviorLayer`, `FilterPipeline`, and `EnrichmentCaps` from `app.state` instead.

## must_haves

- `CoachingService.__init__` accepts `behavior: BehaviorLayer`, `pipeline: FilterPipeline`, `caps: EnrichmentCaps`
- `self._ethos`, `self._boundaries`, `self._allowlist` populated from `behavior.get_*()` not hardcoded paths
- `self._scanner` replaced by `pipeline` — `pipeline.run_input/run_output` called instead of direct `scanner.scan_*`
- `_apply_caps()` reads from `caps` (EnrichmentCaps), not `get_settings().coaching_cap_*`
- `grep -n "PERSONAS_DIR\|coaching_cap_\|SafetyScanner()" be/app/coaching/service.py` returns 0 after changes
- `docker compose run --rm api python -c "from app.coaching.service import CoachingService; print('ok')"` exits 0
