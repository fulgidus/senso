## Spacing Scale

Declared values (all multiples of 4 - mapped to Tailwind v4 spacing utilities):

| Token | Value                  | Usage                                                           |
| ----- | ---------------------- | --------------------------------------------------------------- |
| xs    | 4px (`gap-1`, `p-1`)   | Icon gaps, badge padding, inline chip spacing                   |
| sm    | 8px (`gap-2`, `p-2`)   | Compact element spacing, label-to-value gaps                    |
| md    | 16px (`gap-4`, `p-4`)  | Default element spacing, card inner padding                     |
| lg    | 24px (`gap-6`, `p-6`)  | Section padding (matches existing card pattern `p-6`)           |
| xl    | 32px (`gap-8`, `mt-8`) | Between profile sections (summary → charts → insights)          |
| 2xl   | 48px (`py-12`)         | Page-level vertical rhythm (matches `py-10` on IngestionScreen) |
| 3xl   | 64px                   | Reserved - not used in this phase                               |

**Exceptions:**
- Touch targets for interactive affordance labels (income source badge tap zone): minimum **44px** height.
- Processing screen step indicators: minimum **40px** tap height for "Come back later" button.
- Chart container minimum height: **220px** (Recharts minimum for readable axis labels on mobile).

## Typography

All text uses **Geist Variable** - the single declared font family for both heading and body roles (per `index.css`: `--font-heading: var(--font-sans)`, `--font-sans: 'Geist Variable'`).

| Role    | Size                        | Weight         | Line Height | Usage                                                                                            |
| ------- | --------------------------- | -------------- | ----------- | ------------------------------------------------------------------------------------------------ |
| Body    | 16px (`text-base`)          | 400 (regular)  | 1.5         | Profile section body text, questionnaire labels, insight card educational framing                |
| Label   | 14px (`text-sm`)            | 400 (regular)  | 1.5         | Muted secondary labels, data source badges ("from payslip"), chart axis labels, category tags    |
| Heading | 20px (`text-xl`)            | 600 (semibold) | 1.2         | ProfileScreen section headings (e.g. "Your Spending Breakdown"), processing status title         |
| Display | 28px (`text-2xl font-bold`) | 700 (bold)     | 1.2         | App name "S.E.N.S.O." in header (matches existing IngestionScreen pattern: `text-2xl font-bold`) |

**Constraint:** Maximum 4 type sizes, maximum 2 weights (400 regular + 600 semibold) in new components. The existing `text-2xl font-bold` on the app header is a frozen pattern - carry it forward unchanged.

**Insight card headline:** 16px semibold (600) - visually punchy but not a new size, reuses body size at elevated weight.

## Color

All values sourced from `senso/src/index.css` `:root` (light mode). Dark mode tokens are defined in `.dark` - implementation must use CSS variable references (`bg-background`, `text-foreground`, etc.), never raw hex.

| Role            | Light Value                                                                           | Usage                                                                                             |
| --------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `--background`: `oklch(1 0 0)` - white                                                | Main page surface, `ProcessingScreen` and `ProfileScreen` background                              |
| Secondary (30%) | `--card`: `#DBE2EF` (slate-blue tint)                                                 | Profile summary card, spending breakdown card, insight card backgrounds, questionnaire card shell |
| Accent (10%)    | `--primary`: `#3F72AF` (medium blue)                                                  | See accent reserved list below                                                                    |
| Destructive     | `--destructive`: `#112D4E` (deep navy, light); `oklch(0.704 0.191 22.216)` red (dark) | Destructive actions only (remove/reset profile)                                                   |
| Muted           | `--muted-foreground`: `oklch(0.58 0.031 107.3)`                                       | Source labels ("from payslip"), chart axis labels, secondary metadata                             |
| Success         | `text-green-600` (Tailwind utility)                                                   | "All confirmed ✓" badge (matches existing IngestionScreen pattern)                                |

**Accent (`#3F72AF`) reserved for:**
1. Primary CTA buttons ("View My Profile", "Save Profile", "Continue to Profile") - `variant="default"` (bg-primary)
2. Progress bar fill on `ProcessingScreen` job steps
3. Active/selected state on questionnaire option pills
4. Chart series color 1 (primary bar fill for income or largest category)
5. Income source badge border (to draw attention to the labelled source)

**Accent is NOT used for:** body text, card backgrounds, section headings, muted labels, chart axis lines, or secondary metadata.

**Chart palette:** Use existing `--chart-1` through `--chart-5` (green tonal scale from `oklch(0.845 0.143 164.978)` down to `oklch(0.432 0.095 166.913)`) for category breakdown charts. Maximum 5 visible categories before grouping remainder as "Other".

## Screen Inventory

This phase introduces 3 new screens and extends 1 existing component. All screens follow the `mx-auto w-full max-w-4xl px-6 py-10` layout from `IngestionScreen`.

### 1. ProcessingScreen

**Purpose:** Intermediate loading/status screen shown after confirm-all triggers background categorization. Polls `GET /profile/status`.

**Layout:**
- Centered single-column content, max-width 480px (narrower than full layout - focused task state).
- Top: app header (identical to IngestionScreen - `text-2xl font-bold` "S.E.N.S.O." + email + sign-out).
- Body card (`bg-card`, `rounded-2xl`, `border border-border`, `p-6`):
  - Section title: "Analysing your finances" (20px semibold, `text-foreground`).
  - Status step list: 3-4 items with step indicators (icon: `Loader2` spinning when active, `CheckCircle2` when complete, `Circle` when pending). Steps: "Categorising transactions", "Identifying patterns", "Generating insights", "Building your profile".
  - Brief educational copy below step list: "This usually takes 15-30 seconds." (14px, `text-muted-foreground`).
  - Progress: accent-colored `bg-primary` progress bar (CSS transition, not animated skeleton).
  - "Come back later" link-button (`variant="ghost"`, `text-sm`, `text-muted-foreground`) - navigates back to IngestionScreen, does NOT cancel job.

**States:**
- `queued`: first step spinner active, others pending.
- `categorizing`: first step complete, second spinner active.
- `generating_insights`: first two complete, third active.
- `complete`: all steps complete → auto-advance to ProfileScreen after 500ms delay.
- `failed`: error inline message ("Something went wrong. Your documents are safe - try again.") + "Return to uploads" button.

---

### 2. OnboardingChoiceScreen

**Purpose:** Shown when a user reaches the profile flow with no confirmed uploads. Presents two paths: file-based or questionnaire.

**Layout:**
- Centered, max-width 480px.
- Title: "How should we build your profile?" (20px semibold).
- Two option cards side-by-side (on ≥640px) or stacked (mobile). Each card: `bg-card`, `rounded-2xl`, `border-2 border-border hover:border-primary` (hover lifts border to accent).
  - Card 1: Icon `FileUp` (lucide), heading "From your files", body "Use documents you've already uploaded." CTA: "Use my uploads" (`variant="default"`).
  - Card 2: Icon `MessageSquare` (lucide), heading "Answer a few questions", body "Quick 3-question setup or a fuller 7-10 question profile." CTA: "Start questionnaire" (`variant="outline"`).

---

### 3. QuestionnaireScreen

**Purpose:** Collects income/expense estimates when file data is absent or supplementary.

**Layout:**
- Single question displayed at a time. Progress indicator: "Question 2 of 3" (14px, `text-muted-foreground`) top-right.
- Card shell (`bg-card`, `rounded-2xl`, `border border-border`, `p-6`).
- Question text: 20px semibold.
- Input or option pills (see Interaction Patterns below).
- Navigation row: "Back" (`variant="ghost"`) left, "Next" / "Finish" (`variant="default"`) right.

**Quick questionnaire questions (3):**
1. Employment type - 4 option pills: "Employed", "Self-employed", "Student", "Other".
2. Monthly net income - number input with currency prefix (€ or user-selected currency from Q3).
3. Primary currency - select: EUR, USD, GBP, CHF, other.

**Thorough questionnaire additions (4-7 more):**
4. Fixed monthly costs - number input (€/month estimated).
5. Other income sources - multi-select pills: "Side work", "Rental income", "Investments", "Benefits", "None".
6. Household size - stepper: 1-6+ people.
7. Savings behavior - 3 option pills: "Not saving", "Occasional savings", "Regular savings".
8. Financial goal - 3 option pills: "Save more", "Reduce debt", "Just track spending".

---

### 4. ProfileScreen

**Purpose:** Primary output screen showing the structured financial profile derived from files and/or questionnaire.

**Layout sections (top-to-bottom, `gap-8` between):**

#### Header (matches IngestionScreen pattern)
- "S.E.N.S.O." display text + email + sign-out button.
- Below: "Your Financial Profile" (20px semibold) + subtitle: "Based on [N] documents confirmed [date]" (14px, `text-muted-foreground`).
- "Add more documents" link-button (`variant="ghost"`, lucide `Plus` icon) - navigates to IngestionScreen.

#### Summary Card (Deterministic - no LLM)
- `bg-card`, `rounded-2xl`, `border border-border`, `p-6`.
- Three stat tiles in a row (`grid grid-cols-3 gap-4` on ≥640px, stacked on mobile):
  - **Monthly Income**: large number (20px semibold, `text-foreground`) + source badge below ("from payslip" / "from questionnaire" / "estimated from transactions") in 14px, `text-muted-foreground`, accent-colored border.
  - **Monthly Expenses**: large number + "(recurring)" label.
  - **Available Margin**: large number in accent color (`text-primary`) if positive, `text-destructive` if negative.
- No chart in this tile - numbers only, high scannability.

#### Spending Breakdown (Chart - Deterministic)
- `bg-card`, `rounded-2xl`, `border border-border`, `p-6`.
- Section heading: "Where your money goes" (20px semibold).
- shadcn `<Chart>` wrapping Recharts. Chart type: horizontal bar chart (best for labeled categories with long names).
- Max 5 categories displayed; remainder grouped as "Other". Categories use `--chart-1` through `--chart-5`.
- Below chart: category pill row showing each category label + total. Pills: `bg-secondary`, `rounded-full`, `px-3 py-1`, 14px.
- If only 1 month of data available: bar chart only. If 2+ months: show month-over-month grouped bar or line trend in a second mini-chart.

#### Income vs. Expenses Bar (Chart - Deterministic)
- Only shown if income data is available (not "estimated from transactions" or has a reliable figure).
- shadcn `<Chart>` wrapping Recharts - grouped vertical bar: income (chart-1 green) vs. total expenses (chart-3 mid-green).
- Section heading: "Income vs. Expenses" (20px semibold).
- Single bar pair if one month of data; multi-bar if multiple months.

#### AI Insight Cards (1-3 LLM-generated)
- Section heading: "What your data shows" (20px semibold).
- Each card: `bg-card`, `rounded-2xl`, `border border-border`, `p-5`.
- Card anatomy:
  - Icon: lucide `Lightbulb` (14px, accent color `text-primary`).
  - Headline: 16px semibold - provocative, data-backed (e.g. "Subscriptions are your hidden expense").
  - Data point: 14px regular, `text-muted-foreground` - concrete number (e.g. "€127/month across 6 services").
  - Educational framing: 14px regular, `text-foreground` - 1-2 sentences, jargon-free plain language.
- Cards stacked vertically on mobile, horizontal 3-column grid on ≥1024px.
- Loading state: 3 skeleton cards (pulse animation) while LLM generates.

#### Confirm/Correct Summary (Step before saving)
- Shown as an inline collapsible section OR a separate modal (implementation agent's discretion, but inline preferred for demo speed).
- Heading: "Does this look right?" (20px semibold).
- Editable fields: income figure (number input), monthly expenses (number input), margin (read-only, computed). Each field shows its source badge.
- CTA row: "Save Profile" (`variant="default"`, full-width on mobile) + "Make changes" (`variant="outline"`).
- On save: POST /profile/confirm → success → profile locked, inline success toast ("Profile saved").

## Interaction Patterns

### Option Pills (questionnaire)
- Unselected: `bg-secondary text-secondary-foreground rounded-full px-4 py-2 text-sm border border-border`.
- Selected: `bg-primary text-primary-foreground rounded-full px-4 py-2 text-sm`.
- Keyboard: full focus ring (`focus-visible:ring-3 focus-visible:ring-ring/50` - matches button pattern).

### Polling / Loading States
- `ProcessingScreen`: 3-second initial delay before first poll, then poll every 5 seconds. Show `Loader2` icon spinning (animate-spin) next to active step.
- Chart loading: skeleton rectangles (`bg-muted animate-pulse rounded`) at chart container dimensions.
- Insight card loading: 3 skeleton cards same aspect ratio as real cards.

### Navigation Transitions
- IngestionScreen → ProcessingScreen: triggered immediately on confirm-all success response. No delay.
- ProcessingScreen → ProfileScreen: auto-advance after 500ms on `status === "complete"`.
- ProfileScreen → IngestionScreen: instant, preserve scroll position on return.
- All screen transitions: no animated page transitions (add-on, not required for MVP demo).

### Mobile Responsiveness
- Breakpoint: `sm:` (640px) for 2-column option cards, `lg:` (1024px) for 3-column insight cards.
- All chart containers: `min-h-[220px]` to ensure Recharts renders axes. Full width on mobile.
- Stat tiles: `grid-cols-1 sm:grid-cols-3` - stack on mobile.

## Copywriting Contract

| Element                                    | Copy                                                                                                                |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| Primary CTA - profile entry                | "View My Profile"                                                                                                   |
| Primary CTA - save confirmed profile       | "Save Profile"                                                                                                      |
| Primary CTA - questionnaire advance        | "Next" (steps 1-N-1) / "Finish" (last step)                                                                         |
| Processing screen title                    | "Analysing your finances"                                                                                           |
| Processing screen subtitle                 | "This usually takes 15-30 seconds."                                                                                 |
| Processing fallback link                   | "Come back later"                                                                                                   |
| Profile screen headline                    | "Your Financial Profile"                                                                                            |
| Profile screen subtitle                    | "Based on [N] document(s) confirmed [date]"                                                                         |
| Summary section heading                    | (no heading - stats speak for themselves)                                                                           |
| Spending breakdown heading                 | "Where your money goes"                                                                                             |
| Income vs. expenses heading                | "Income vs. Expenses"                                                                                               |
| Insight cards heading                      | "What your data shows"                                                                                              |
| Confirm/correct heading                    | "Does this look right?"                                                                                             |
| Income source - payslip                    | "from payslip"                                                                                                      |
| Income source - questionnaire              | "from questionnaire"                                                                                                |
| Income source - estimated                  | "estimated from transactions"                                                                                       |
| Empty state - no uploads, no questionnaire | "No data yet. Upload a document or answer a few questions to build your profile."                                   |
| Empty state - no categories                | "We couldn't categorise your transactions yet. Add more documents to improve accuracy."                             |
| Error - processing failed                  | "Something went wrong. Your documents are safe - try again."                                                        |
| Error - profile load failed                | "Couldn't load your profile. Check your connection and refresh."                                                    |
| Error - save failed                        | "Profile not saved. Check your connection and try again."                                                           |
| Positive margin label                      | "Available this month"                                                                                              |
| Negative margin label                      | "Over budget this month"                                                                                            |
| Destructive - reset profile                | "Reset Profile": "This will delete your current profile and insights. Your uploaded documents will stay. Continue?" |

**Language rules (audience: 18-30, low financial literacy):**
- No jargon: "margin" in UI copy = "available this month" / "left over". Never "liquidity" or "net surplus".
- Action-oriented CTAs: verb first ("Save Profile", not "Profile Save").
- Error copy always includes what to do next.
- Avoid exclamation marks except in empty-state encouragement contexts.
