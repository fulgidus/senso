# Plan 20-05 — Prompt Architecture Refactor: Tool-First, Prompt-Lean

## Objective

Refactor the CoachingService system prompt to remove the static profile blob and trust
the tool suite to provide data on demand. The system prompt should encode persona soul,
tool instructions, response schema, and safety rules only — not user data.

---

<task id="20-05-01" title="Remove static profile block from system prompt">
  <objective>
    Delete the `context_block.j2` injection from `_build_system_prompt()`. The coach
    now fetches profile data via `get_user_profile` tool when needed. Update the system
    prompt to tell the coach about available tools and when to use them.
  </objective>
  <read_first>
    - api/app/coaching/service.py — _build_system_prompt(), _fetch_profile_context()
    - api/app/coaching/prompts/context_block.j2 — current profile block template
    - api/app/coaching/prompts/system_base.j2 — base system prompt
  </read_first>
  <action>
    Replace the profile injection section in `system_base.j2` with a tool-use guide:
    ```jinja2
    ## Strumenti disponibili
    Usa gli strumenti per ottenere dati aggiornati dell'utente. Non inventare cifre.

    - `get_user_profile()` — snapshot finanziario: reddito, spese, tasso risparmio
    - `search_user_transactions(query, date_from?, date_to?, category?)` — cerca nelle sue transazioni
    - `get_timeline_events(types?)` — eventi di vita finanziaria (trasloco, aumento stipendio, ecc.)
    - `get_user_preferences()` — obiettivi, abitudini positive, regole personali
    - `recall_past_insights(topic)` — ricorda cosa sai già di questo utente
    - `search_content(query, locale)` — articoli, video, slide educative
    - `search_italy_rules(topic)` — regole fiscali italiane (IRPEF, INPS, bonus, ecc.)

    Regola: chiama `get_user_profile()` all'inizio di ogni conversazione finanziaria.
    Non includere dati dell'utente nella risposta se non li hai ottenuti da uno strumento.
    ```

    Remove `context_block.j2` from the prompt assembly call.
    Keep `timeline_block.j2` as a brief "heads up" hint (3 events max, not full data).
  </action>
  <acceptance_criteria>
    - `grep "context_block" api/app/coaching/service.py` → 0 matches
    - System prompt character count ≤ 60% of current (measure before/after)
    - `test_coaching_service.py::test_system_prompt_no_profile_blob` passes
    - No regression: coach still answers correctly for a user with profile data
      (verified by calling `get_user_profile` tool in integration test)
  </acceptance_criteria>
</task>
