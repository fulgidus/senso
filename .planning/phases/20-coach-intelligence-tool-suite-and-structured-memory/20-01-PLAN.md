# Plan 20-01 - Italy Rules Knowledge Base + search_italy_rules Tool

## Objective

Build a static Italian financial rules knowledge base and expose it as an LLM tool,
giving the coach authoritative, structured access to IRPEF, INPS, bonus cultura, 730,
TFR, pension, and ISEE rules without hallucinating percentages and thresholds.

---

<task id="20-01-01" title="Create italy_rules.json knowledge base">
  <objective>
    Write `api/app/content/italy_rules.json` with structured entries for Italian
    financial rules relevant to young adults (18-30). Each entry has id, locale,
    type, title, topics[], summary, details.
  </objective>
  <read_first>
    - api/app/content/ - existing catalog structure (articles.json, videos.json)
  </read_first>
  <action>
    Create with at least these categories (20+ entries):
    - **IRPEF**: 2024 brackets (23%, 35%, 43%), detrazioni per lavoro dipendente
    - **INPS**: aliquote contributive (9.19% dipendenti), minimale/massimale
    - **Bonus**: bonus cultura (18app), bonus psicologo, bonus affitti giovani, bonus mobili
    - **730/dichiarazione**: scadenze, chi deve farlo, rimborso IRPEF timing
    - **TFR**: cos'è, quando si matura, fondo pensione vs liquidazione diretta
    - **ISEE**: cos'è, calcolo, soglie per bonus, quando serve
    - **Partita IVA**: regime forfettario (15%/5%), contributi fissi INPS artigiani/commercianti
    - **Affitti**: cedolare secca (21%/10%), detrazioni inquilino, canone concordato

    Format per entry:
    ```json
    {
      "id": "irpef-brackets-2024",
      "locale": "it",
      "type": "rule",
      "title": "Scaglioni IRPEF 2024",
      "topics": ["irpef", "tasse", "dichiarazione"],
      "summary": "L'IRPEF 2024 ha 3 scaglioni: 23% fino a 28.000€, 35% da 28.000€ a 50.000€, 43% oltre.",
      "details": "..."
    }
    ```
  </action>
  <acceptance_criteria>
    - File has ≥20 entries covering all categories above
    - All entries have `locale: "it"` (en mirror entries optional for MVP)
    - `json.loads(open("italy_rules.json").read())` succeeds
  </acceptance_criteria>
</task>
