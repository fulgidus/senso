# Plan 20-02 — get_user_profile + search_user_transactions Tools

## Objective

Give the coach tools to access the user's own financial data on demand: a profile
snapshot and a BM25 search over personal transaction descriptions.

---

<task id="20-02-01" title="get_user_profile tool">
  <objective>
    Implement `get_user_profile()` as an LLM tool that returns a condensed financial
    snapshot: income range, monthly expense estimate, top 3 spending categories,
    savings rate, net worth estimate, and verified income sources.
  </objective>
  <read_first>
    - api/app/services/profile_service.py — ProfileService, get_profile_dto()
    - api/app/coaching/service.py — existing profile context in _fetch_profile_context()
  </read_first>
  <action>
    ```python
    _GET_USER_PROFILE_TOOL = {
        "type": "function",
        "function": {
            "name": "get_user_profile",
            "description": "Get the user's current financial profile snapshot: income, expenses, savings rate, top spending categories, and verified income sources.",
            "parameters": {"type": "object", "properties": {}, "required": [],
                           "additionalProperties": False}
        }
    }
    ```

    In `_tool_executor`:
    ```python
    if name == "get_user_profile":
        profile = ProfileService(self.db).get_profile_dto(self.user_id)
        return {
            "monthly_income_range": [profile.monthly_income_min, profile.monthly_income_max],
            "monthly_expenses_estimate": profile.monthly_expense_avg,
            "savings_rate_pct": profile.savings_rate,
            "top_categories": profile.top_categories[:3],
            "net_worth_estimate": profile.net_worth_estimate,
            "verified_income_sources": profile.verified_income_sources or [],
        }
    ```
  </action>
  <acceptance_criteria>
    - Tool returns dict with all listed keys
    - None values for uncalculated fields (not errors)
    - `test_coaching_tools.py::test_get_user_profile_tool` passes
  </acceptance_criteria>
</task>
