# Plan 20-04 - Structured Coaching Memory + recall_past_insights Tool

## Objective

Migrate `coaching_insights` from an unstructured list to a typed, deduplicated, queryable
memory store. Add `recall_past_insights` as an LLM tool and implement automatic pruning
and deduplication.

---

<task id="20-04-01" title="Migrate coaching_insights to structured format">
  <objective>
    Change `coaching_insights` entries from free-form dicts to a typed structure:
    `{topic, value, created_at, updated_at, expires_at, source_session_id}`.
    Migrate existing entries: wrap free-form dict in `value`, infer `topic` from keys.
  </objective>
  <read_first>
    - api/app/db/models.py - UserProfile.coaching_insights
    - api/app/coaching/service.py - _persist_coaching_insight()
  </read_first>
  <action>
    New insight structure:
    ```python
    @dataclass
    class CoachingInsight:
        topic: str       # short label: "savings_rate", "rent_budget", "emergency_fund"
        value: str       # human-readable fact: "risparmia circa il 12% del reddito mensile"
        created_at: str  # ISO datetime
        updated_at: str  # ISO datetime
        expires_at: str | None  # ISO datetime, None = permanent
        source_session_id: str | None
    ```

    Update `_persist_coaching_insight()`:
    - Require `topic` and `value` fields from LLM (update coaching_response.schema.json)
    - Before appending: check if insight with same `topic` exists → update `value` + `updated_at`
    - Prune insights with `expires_at < now()` on load
    - Auto-prune insights older than 180 days (no expires_at and created_at > 180 days)

    Backfill migration: one-time script in `_add_missing_columns()` that rewraps existing
    free-form insights into the new structure.
  </action>
  <acceptance_criteria>
    - `coaching_insights` entries all have `topic`, `value`, `created_at`
    - Same topic insight → `updated_at` updated, no duplicate entry
    - Insights >180 days old → pruned on next load
    - `pytest test_coaching_service.py::test_insight_dedup` passes
  </acceptance_criteria>
</task>

---

<task id="20-04-02" title="recall_past_insights LLM tool">
  <objective>
    Register `recall_past_insights(topic)` as an LLM tool that BM25-searches the
    user's structured coaching insights by topic keyword.
  </objective>
  <read_first>
    - api/app/coaching/service.py - _tool_executor(), existing tools
  </read_first>
  <action>
    ```python
    _RECALL_INSIGHTS_TOOL = {
        "type": "function",
        "function": {
            "name": "recall_past_insights",
            "description": "Recall previously captured insights about this user's financial behaviour, goals, or context from past conversations.",
            "parameters": {
                "type": "object",
                "properties": {
                    "topic": {"type": "string",
                              "description": "Keyword to search by (e.g. 'risparmio', 'affitto', 'spese fisse')"}
                },
                "required": ["topic"],
                "additionalProperties": False
            }
        }
    }
    ```
    In `_tool_executor`:
    - Load `coaching_insights` from profile
    - BM25 over `topic + " " + value`
    - Return top 5 matches as `[{topic, value, created_at}]`
  </action>
  <acceptance_criteria>
    - Tool returns relevant insights for topic query
    - Empty insights → returns [] without error
    - `test_coaching_tools.py::test_recall_past_insights` passes
  </acceptance_criteria>
</task>

<task id="20-04-03" title="Update coaching_response.schema.json to require topic in new_insight">
  <objective>
    Enforce that the LLM's `new_insight` field in `coaching_response.schema.json` has
    both `topic` and `value` string fields (not a free-form dict), with `additionalProperties: false`.
  </objective>
  <read_first>
    - api/app/coaching/schemas/coaching_response.schema.json
  </read_first>
  <action>
    Update `new_insight` schema from `{"type": "object"}` to:
    ```json
    "new_insight": {
      "oneOf": [
        {"type": "null"},
        {
          "type": "object",
          "properties": {
            "topic": {"type": "string", "maxLength": 50},
            "value": {"type": "string", "maxLength": 200},
            "expires_days": {"type": "integer", "description": "Optional: days until this insight expires"}
          },
          "required": ["topic", "value"],
          "additionalProperties": false
        }
      ]
    }
    ```
    Update Jinja2 system prompt to show LLM the new required shape.
  </action>
  <acceptance_criteria>
    - Schema validates correctly for `{topic: "...", value: "..."}` and `null`
    - Schema rejects `{}` or `{topic_only: "..."}` (jsonschema.validate raises)
    - `pytest test_coaching_cards.py` passes (no schema regression)
  </acceptance_criteria>
</task>
