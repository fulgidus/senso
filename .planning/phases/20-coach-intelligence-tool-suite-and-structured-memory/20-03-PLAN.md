# Plan 20-03 - User Preferences (Goals, Dos, Don'ts) + get_user_preferences Tool

## Objective

Add structured user preferences (financial goals, spending dos, spending don'ts) to the
profile, expose them via LLM tool, and provide a minimal frontend editor.

<task id="20-03-03" title="Frontend preferences editor in Settings">
  <objective>
    Add a "Preferenze finanziarie" section to SettingsScreen (or ProfileScreen) with
    three simple tag-input lists for goals, dos, don'ts. Tags are short strings, add
    via Enter, remove via ×.
  </objective>
  <read_first>
    - senso/src/features/settings/SettingsScreen.tsx - existing settings sections
    - senso/src/locales/it.json - add preference keys
  </read_first>
  <action>
    Simple tag input component:
    ```tsx
    function TagList({ label, items, onChange }: TagListProps) {
      const [input, setInput] = useState("")
      return (
        <div>
          <label>{label}</label>
          <div className="flex flex-wrap gap-1 mt-1">
            {items.map(item => (
              <span key={item} className="badge">
                {item}
                <button onClick={() => onChange(items.filter(i => i !== item))}>×</button>
              </span>
            ))}
          </div>
          <input value={input} onChange={e => setInput(e.target.value)}
                 onKeyDown={e => { if (e.key === "Enter" && input.trim()) {
                   onChange([...items, input.trim()]); setInput("") }}} />
        </div>
      )
    }
    ```
    Auto-save on change (debounced PATCH).
  </action>
  <acceptance_criteria>
    - User can add/remove goals, dos, donts via tag UI
    - Changes persisted via PATCH `/profile/preferences`
    - i18n keys present
    - `pnpm build` clean
  </acceptance_criteria>
</task>
