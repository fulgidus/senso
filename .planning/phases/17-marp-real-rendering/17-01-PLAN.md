# Plan 17-01 — Install Marp Core + Rewrite Renderer + Register Themes

deck in one call, inject scoped MARP CSS, and register `senso-light` / `senso-dark` themes

<task id="17-01-03" title="Rewrite MarpSlideViewer rendering logic">
  <objective>
    Replace `marked.parse()` + manual `parseSlides()` with `new Marp({ ...}).render(raw)`.
    MARP handles front-matter stripping, slide splitting, and HTML+CSS generation in one call.
    Inject the returned CSS into a `<style>` tag scoped to the viewer container.
    Remove the `marked` import and dependency.
  </objective>
  <read_first>
    - senso/src/features/coaching/MarpSlideViewer.tsx — current implementation
    - senso/src/content/slideIndex.ts — slide deck IDs and raw imports
  </read_first>
  <action>
    Rewrite the rendering section of `MarpSlideViewer.tsx`:

    ```typescript
    import { Marp } from "@marp-team/marp-core"
    // remove: import { marked } from "marked"

    function useMarpRender(raw: string, theme: "senso-light" | "senso-dark") {
      return useMemo(() => {
        const marp = new Marp({ html: false })
        marp.themeSet.add(sensoLightTheme)
        marp.themeSet.add(sensoDarkTheme)
        // Inject theme into front-matter if not already specified
        const src = raw.startsWith("---") ? raw : `---\ntheme: ${theme}\n---\n${raw}`
        const { html, css, comments } = marp.render(src)
        // html contains all slides as <section> elements
        const parser = new DOMParser()
        const doc = parser.parseFromString(html, "text/html")
        const sections = Array.from(doc.querySelectorAll("section"))
        return { slides: sections.map(s => s.outerHTML), css, total: sections.length }
      }, [raw, theme])
    }
    ```

    Render each slide via `dangerouslySetInnerHTML` (same as before).
    Inject `css` into a `<style>` tag using `useEffect` (append to document head,
    clean up on unmount using a unique `data-marp-deck-id` attribute to avoid duplicates).
  </action>
  <acceptance_criteria>
    - `grep "marked" senso/src/features/coaching/MarpSlideViewer.tsx` → 0 matches
    - Slide viewer renders `<section>` elements from MARP output
    - Slide count matches `---` separator count in source files
    - No visible `<!--` text in rendered output
    - `pnpm build` clean
  </acceptance_criteria>
</task>
