# Plan 17-02 — Fullscreen Portal + Keyboard Nav + Unit Tests

## Objective

Move fullscreen rendering to a `ReactDOM.createPortal` (renders into `document.body`),
ensure keyboard navigation (arrow keys, Escape, F) works, and add vitest unit tests
that verify slide parsing and render counts for all decks in `slideIndex.ts`.

---

<task id="17-02-01" title="Fullscreen via React portal">
  <objective>
    Replace the `marp-fullscreen-overlay` CSS overlay with a proper React portal
    that appends to `document.body`. This escapes any `overflow: hidden` or `z-index`
    stacking context in parent components (e.g. coach message bubbles, admin drawer).
  </objective>
  <read_first>
    - senso/src/features/coaching/MarpSlideViewer.tsx — current fullscreen implementation
  </read_first>
  <action>
    ```typescript
    import { createPortal } from "react-dom"

    // In MarpSlideViewer, replace the conditional fullscreen div with:
    {fullscreen && createPortal(
      <div
        className="fixed inset-0 z-[9999] bg-black/90 flex items-center justify-center"
        onClick={() => setFullscreen(false)}
        role="dialog"
        aria-modal="true"
        aria-label={`Slide deck fullscreen: ${title}`}
      >
        <div
          className="w-full max-w-4xl aspect-[16/9] mx-4"
          onClick={e => e.stopPropagation()}
        >
          {/* slide rendered at full size */}
        </div>
      </div>,
      document.body
    )}
    ```
    Ensure `useEffect` locks body scroll (`document.body.style.overflow = "hidden"`)
    while fullscreen is active and restores it on close.
  </action>
  <acceptance_criteria>
    - Fullscreen renders above the nav drawer and all modals
    - Body scroll is locked while fullscreen active
    - Clicking outside the slide closes fullscreen
    - Escape key closes fullscreen
  </acceptance_criteria>
</task>

<task id="17-02-03" title="Vitest unit tests for MARP rendering">
  <objective>
    Add unit tests that import each slide deck from `SLIDE_INDEX`, render it with
    the new MARP-based renderer, and assert:
    - Slide count ≥ 1
    - No visible `<!--` or `---` separator text in rendered HTML
    - Theme CSS is non-empty
  </objective>
  <read_first>
    - senso/src/content/slideIndex.ts — deck IDs and raw content
    - senso/vitest.config.ts or vite.config.ts — test setup
  </read_first>
  <action>
    Create `senso/src/features/coaching/__tests__/MarpSlideViewer.test.ts`:
    ```typescript
    import { describe, it, expect } from "vitest"
    import { SLIDE_INDEX } from "@/content/slideIndex"
    // import the render helper (extracted pure function from component)

    describe("MARP rendering", () => {
      Object.entries(SLIDE_INDEX).forEach(([id, raw]) => {
        it(`deck ${id} renders ≥1 slides without comment leakage`, () => {
          const { slides, css } = renderMarpDeck(raw, "senso-light")
          expect(slides.length).toBeGreaterThan(0)
          expect(slides.join("")).not.toContain("<!--")
          expect(css.length).toBeGreaterThan(0)
        })
      })
    })
    ```
    Extract `renderMarpDeck(raw, theme)` as a pure function from `MarpSlideViewer.tsx`
    so it can be imported in tests without DOM.
  </action>
  <acceptance_criteria>
    - `docker compose run --rm frontend pnpm test` passes all new tests
    - One test case per deck in SLIDE_INDEX
  </acceptance_criteria>
</task>
