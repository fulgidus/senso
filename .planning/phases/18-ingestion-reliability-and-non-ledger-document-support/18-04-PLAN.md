# Plan 18-04 — Receipt Module + Unified enrich_from_extraction()

## Objective

Build the receipt extractor and unify all profile enrichment paths behind a single
`ProfileService.enrich_from_extraction()` dispatcher so the ingestion service has
one call site regardless of document type.

---

<task id="18-04-01" title="Create receipt_it builtin module">
  <objective>
    Write `api/app/ingestion/modules/builtin/receipt_it.py`. Handles Italian
    receipts (scontrino fiscale, ricevuta). Extracts merchant, total amount,
    individual items if line-item text is available, and date.
  </objective>
  <read_first>
    - api/app/schemas/ingestion.py — receipt fields (merchant, total_amount, items, date)
  </read_first>
  <action>
    ```python
    FINGERPRINT = ["scontrino", "ricevuta fiscale", "totale", "importo", "p.iva"]
    MIME_TYPES = ["application/pdf", "image/jpeg", "image/png"]
    MODULE_VERSION = "1.0.0"
    ```
    For image receipts, `raw_text` comes from liteparse OCR. Extract:
    - Merchant: last non-numeric line before items, or header
    - Total: `"TOTALE\s+([\d,.]+)"` (case insensitive)
    - Items: lines matching `<desc>\s+<price>` pattern (best-effort, empty list ok)
    - Date: ISO or Italian date pattern in header/footer
  </action>
  <acceptance_criteria>
    - Text receipt fixture → merchant, total_amount, date extracted
    - `document_type = "receipt"`, `transactions = []`
  </acceptance_criteria>
</task>

---

<task id="18-04-02" title="Unified enrich_from_extraction() dispatcher">
  <objective>
    Add `ProfileService.enrich_from_extraction(user_id, extraction: ExtractedDocument)`
    that dispatches to the correct enrichment method based on `extraction.document_type`.
    Replace the per-type call sites in IngestionService with this single call.
  </objective>
  <read_first>
    - api/app/services/profile_service.py — existing enrichment methods
    - api/app/services/ingestion_service.py — extraction completion handler
  </read_first>
  <action>
    ```python
    def enrich_from_extraction(self, user_id: str, extraction: ExtractedDocument) -> None:
        dispatch = {
            "payslip": self.enrich_from_payslip,
            "utility_bill": self.enrich_from_utility_bill,
            "invoice": self.enrich_from_invoice,
            "receipt": self.enrich_from_receipt,
        }
        fn = dispatch.get(extraction.document_type)
        if fn:
            fn(user_id, extraction)
        # bank_statement, unknown: no enrichment (handled via transactions)
    ```

    In `IngestionService._run_extraction_background()`, after extraction completes:
    ```python
    profile_service.enrich_from_extraction(user_id, result.extraction)
    ```
  </action>
  <acceptance_criteria>
    - Single call in IngestionService dispatches to correct enrichment
    - `bank_statement` and `unknown` types → no error, no enrichment
    - All 4 non-ledger types → correct profile column updated
  </acceptance_criteria>
</task>
