# Plan 18-02 — Payslip Builtin Module + Profile Income Enrichment

## Objective

Build the Italian payslip (busta paga / cedolino) builtin extraction module and wire
its output into `user_profiles.verified_income_sources`.

---

<task id="18-02-01" title="Create payslip_it builtin module">
  <objective>
    Write `api/app/ingestion/modules/builtin/payslip_it.py`. Extract net salary,
    gross salary, employer, pay period, INPS contribution, and IRPEF withheld from
    Italian payslip PDFs (text layer or OCR).
  </objective>
  <read_first>
    - api/app/ingestion/modules/builtin/fineco_it.py — module structure reference
    - api/app/schemas/ingestion.py — ExtractedDocument payslip fields
  </read_first>
  <action>
    ```python
    FINGERPRINT = ["busta paga", "cedolino", "netto", "lordo", "datore di lavoro"]
    MIME_TYPES = ["application/pdf"]
    MODULE_VERSION = "1.0.0"

    def extract(file_path: Path, raw_text: str) -> ExtractedDocument:
        # Regex-based extraction for common Italian payslip layouts
        # Patterns: "Netto a Pagare\s+[\d,.]+", "Retribuzione Lorda\s+[\d,.]+"
        # Employer: "Datore di Lavoro[:\s]+(.+)"
        # Period: "Mese di (\w+) (\d{4})"
        return ExtractedDocument(
            document_type="payslip",
            net_salary=...,
            gross_salary=...,
            employer=...,
            pay_period=...,
            inps_contribution=...,
            irpef_withheld=...,
            transactions=[],  # payslips have no transaction list
        )
    ```
  </action>
  <acceptance_criteria>
    - Module loads without error (fingerprint + mime_types + extract present)
    - `extract(fixture_payslip.pdf, raw_text)` returns ExtractedDocument with
      `document_type="payslip"` and non-null `net_salary`
  </acceptance_criteria>
</task>
