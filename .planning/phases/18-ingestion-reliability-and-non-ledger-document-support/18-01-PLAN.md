# Plan 18-01 — Content-Hash Dedup + MIME-First Module Routing

## Objective

Add file-level deduplication (sha256 content hash stored on upload) and fix the
ModuleRegistry to use MIME type as the primary routing signal before keyword scanning —
including extracting actual cell text from XLSX files for keyword matching.

---

<task id="18-01-01" title="Add content_hash to uploads table">
  <objective>
    Add a `content_hash VARCHAR(64)` column to the `uploads` table. On each upload,
    compute `sha256(file_bytes).hexdigest()` and check if a row with the same
    `(user_id, content_hash)` already exists. If so, return 409 with `code: "duplicate_file"`.
  </objective>
  <read_first>
    - api/app/db/models.py — Upload model
    - api/app/db/session.py — _add_missing_columns() migration pattern
    - api/app/services/ingestion_service.py — upload_file() method
  </read_first>
  <action>
    1. Add `content_hash = Column(String(64), nullable=True, index=True)` to Upload model
    2. Add migration in `_add_missing_columns()`:
       `ALTER TABLE uploads ADD COLUMN content_hash VARCHAR(64)`
    3. In `IngestionService.upload_file()`, before MinIO put:
       ```python
       import hashlib
       content_hash = hashlib.sha256(file_bytes).hexdigest()
       existing = db.query(Upload).filter_by(user_id=user_id, content_hash=content_hash).first()
       if existing:
           raise IngestionError("duplicate_file", "File already uploaded", status_code=409)
       ```
    4. Set `upload.content_hash = content_hash` when creating the row
  </action>
  <acceptance_criteria>
    - Second upload of same file returns HTTP 409 `{"code": "duplicate_file"}`
    - First upload succeeds and `uploads.content_hash` is populated
    - Pytest: `test_ingestion_dedup.py::test_duplicate_rejected`
  </acceptance_criteria>
</task>

<task id="18-01-02" title="MIME-first routing + XLSX text extraction for fingerprint scan">
  <objective>
    Modify `ModuleRegistry.find_best_match()` to pre-filter candidate modules by MIME
    type before keyword scanning. For XLSX files, extract actual cell text from the
    first sheet (using openpyxl, already a dependency) and run keyword matching on that
    text rather than the raw ZIP bytes.
  </objective>
  <read_first>
    - api/app/ingestion/registry.py — find_best_match(), ModuleEntry
    - api/app/ingestion/modules/builtin/fineco_it.py — FINGERPRINT, MODULE_VERSION
  </read_first>
  <action>
    Add `MIME_TYPES: list[str]` attribute to each builtin module (e.g. fineco_it:
    `MIME_TYPES = ["application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"]`).

    In `ModuleRegistry`:
    ```python
    def find_best_match(self, file_path: Path, mime_type: str) -> ModuleEntry | None:
        candidates = [m for m in self.modules
                      if not getattr(m, 'mime_types', None)
                      or mime_type in m.mime_types]
        scan_text = self._get_scan_text(file_path, mime_type)
        # existing keyword scoring on scan_text
    ```

    `_get_scan_text(file_path, mime_type)`:
    - XLSX → open with openpyxl, read first 200 cells as joined string
    - PDF/text → read first FINGERPRINT_SCAN_BYTES bytes decoded as utf-8 (existing)
    - Other binary → empty string (falls through to adaptive)
  </action>
  <acceptance_criteria>
    - Fineco XLSX file matches `fineco_it` module (score ≥ 0.3) when tested with fixture
    - Non-XLSX file does not enter XLSX module scoring
    - Pytest: `test_ingestion_dedup.py::test_xlsx_fingerprint_uses_cell_text`
  </acceptance_criteria>
</task>
