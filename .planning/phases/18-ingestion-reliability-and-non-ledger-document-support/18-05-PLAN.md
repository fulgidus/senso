<task id="18-05-01" title="Fix non-ledger success condition in adaptive pipeline">
  <objective>
    In `run_adaptive_pipeline()`, change the success check from
    `extraction.transactions or extraction.document_type != "unknown"`
    to also count non-ledger field population as success.
  </objective>
  <read_first>
    - api/app/ingestion/adaptive.py — lines around "if extraction.transactions"
    - api/app/schemas/ingestion.py — non-ledger fields (net_salary, provider, etc.)
  </read_first>
  <action>
    Extract helper:
    ```python
    def _extraction_has_content(ext: ExtractedDocument) -> bool:
        if ext.transactions:
            return True
        non_ledger_types = {"payslip", "receipt", "invoice", "utility_bill"}
        if ext.document_type in non_ledger_types:
            # At least one non-ledger field populated
            return any([ext.net_salary, ext.provider, ext.vendor, ext.merchant])
        return False
    ```
    Replace all `if extraction.transactions or extraction.document_type != "unknown":` 
    with `if _extraction_has_content(extraction):`
  </action>
  <acceptance_criteria>
    - Payslip with 0 transactions → `tier_used != "adaptive"` (matched builtin)
    - Pipeline confidence ≥ 0.7 for builtin-matched non-ledger docs
  </acceptance_criteria>
</task>
