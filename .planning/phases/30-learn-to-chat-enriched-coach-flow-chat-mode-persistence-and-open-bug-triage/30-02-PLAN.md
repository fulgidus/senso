# Plan 30-02: STT Firefox Fix + Chat Mode localStorage Persistence + Guard Test

## Goal

Three targeted fixes:

1. **STT Firefox/LibreWolf**: `useVoiceInput.ts` initializes `backend` state via `useEffect`, meaning the first render always has `backend="none"` → `isSttAvailable=false` → mic button is disabled until the effect fires. On Chrome this flashes briefly. On LibreWolf (no `SpeechRecognition`) this is more visible and can appear broken. Fix: lazy initial state `useState(() => detectBackend())`. Also: ElevenLabs Scribe STT doesn't support `audio/ogg` (Firefox's MediaRecorder default format) — backend must strip codec params from content-type before calling ElevenLabs.

2. **Chat mode persistence**: `useVoiceMode.ts` uses `useState(false)` for `isVoiceMode` — resets to text mode on every navigation. Persist via `localStorage`.

3. **STT guard regression test**: unit test confirming `startRecording()` while already recording is a no-op (todo #27 fix guard).

## must_haves

- `useVoiceInput.ts` uses lazy initial state: `useState<VoiceBackend>(() => detectBackend())` not `useState("none")` + useEffect
- `useVoiceInput.ts` removes the `useEffect(() => setBackend(detectBackend()), [])` call (no longer needed)
- Backend `/coaching/stt` strips codec parameters from content-type before passing to ElevenLabs (e.g. `"audio/ogg;codecs=opus"` → `"audio/ogg"`)
- `useVoiceMode.ts` reads initial `isVoiceMode` from `localStorage.getItem("senso:chatMode") === "voice"`
- `useVoiceMode.ts` writes `localStorage.setItem("senso:chatMode", ...)` when mode toggles
- `useVoiceInput.test.ts` has a test confirming `startRecording()` while `isRecording=true` does NOT call `recognition.start()` again
- `pnpm --prefix senso tsc --noEmit` exits 0
- `python3 -c "import ast; ast.parse(open('api/app/api/coaching.py').read()); print('ok')"` exits 0

```python
# ElevenLabs Scribe doesn't support codec parameters or ogg format natively.
# Normalize: strip codec params (e.g. "audio/ogg;codecs=opus" → "audio/ogg"),
# then remap ogg → webm (ElevenLabs supported list: mp3, mp4, mpeg, mpga, m4a, wav, webm).
_base_ct = raw_content_type.split(";")[0].strip()
if "ogg" in _base_ct:
    # Ogg is not in ElevenLabs supported list — remap to webm for API compatibility.
    # The audio bytes are still valid opus-encoded data; ElevenLabs handles this.
    content_type = "audio/webm"
    filename = "audio.webm" if filename.endswith(".ogg") else filename
else:
    content_type = _base_ct
result = client.speech_to_text.convert(
    audio=(filename, io.BytesIO(audio_bytes), content_type),
```
