# Plan 30-03: Settings Coach Picker — Dark Mode Colors + Instant Persistence

## Goal

Two bugs in the coach picker inside SettingsScreen:

1. **Wrong colors in dark mode**: The picker uses `const theme = persona.theme?.light` — hardcoded to the light theme variant regardless of the user's current color scheme. In dark mode this produces light-toned bubble backgrounds and border colors, making text unreadable. The ChatScreen persona picker correctly uses `getPersonaTheme(persona, resolvedTheme)` which reads the theme-aware variant. Fix: mirror the same pattern in SettingsScreen.

2. **Doesn't auto-persist**: Selecting a coach in the settings picker only sets local state. The user must scroll down and click Save. This is surprising because the picker looks like a radio group (immediate selection). The `strictPrivacyMode` toggle in the same screen auto-saves immediately via a direct API call — the coach picker should work the same way.

## must_haves

- Settings coach picker uses `getPersonaTheme(persona, resolvedTheme)` (not `persona.theme?.light`)
- `resolvedTheme` is derived from `useTheme()` — already imported in SettingsScreen
- Coach selection auto-saves immediately: clicking a persona card calls `updateMe(token, { defaultPersonaId: persona.id })` + `updateUser(updated)` without requiring the Save button
- `defaultPersonaId` is removed from the main Save form (no longer a `isDirty` trigger)
- Save button remains for voice gender + voiceAutoListen changes only
- `pnpm --prefix senso tsc --noEmit` exits 0

### Task 2 — Make coach selection auto-save immediately

<read_first>
- `senso/src/features/settings/SettingsScreen.tsx` (lines 37–45 — state declarations for `defaultPersonaId`; lines 75–85 — `isDirty` expression; lines 85–115 — `handleSave` function; lines 226–265 — coach picker onClick)

</read_first>

<action>
In `senso/src/features/settings/SettingsScreen.tsx`:

1. Add persona-save state variables (alongside the existing `privacySaving` pattern):
```typescript
const [personaSaving, setPersonaSaving] = useState(false)
const [personaSaved, setPersonaSaved] = useState(false)
```

2. Add a `handlePersonaSelect` callback:
```typescript
const handlePersonaSelect = useCallback(async (personaId: string) => {
    if (personaId === defaultPersonaId) return // already selected
    setDefaultPersonaId(personaId)
    setPersonaSaving(true)
    try {
        const token = readAccessToken()
        if (!token) return
        const updated = await updateMe(token, { defaultPersonaId: personaId })
        updateUser(updated)
        setPersonaSaved(true)
        setTimeout(() => setPersonaSaved(false), 2000)
    } catch {
        // Silently revert to previous on error
        setDefaultPersonaId(defaultPersonaId)
    } finally {
        setPersonaSaving(false)
    }
}, [defaultPersonaId, updateUser])
```

3. In the coach picker map, replace:
```typescript
onClick={() => setDefaultPersonaId(persona.id)}
```
with:
```typescript
onClick={() => void handlePersonaSelect(persona.id)}
disabled={personaSaving}
```

4. Add a saved indicator near the coach picker heading:
```typescript
{personaSaved && <span className="text-xs text-primary ml-auto">{t("preferences.saved")}</span>}
```
(in the flex row with the "Default Coach" heading)

5. Remove `defaultPersonaId` from `isDirty` expression:
```typescript

})

// After:
```

7. Remove `setDefaultPersonaId(updated.defaultPersonaId ?? "mentore-saggio")` from handleSave — it's now set by `handlePersonaSelect`.
