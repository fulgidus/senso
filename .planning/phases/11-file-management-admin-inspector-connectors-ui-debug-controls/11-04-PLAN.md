    Create api/app/api/debug.py:

    - Import `require_tester` from `app.api.admin` (added in Plan 01)
    - Import ORM models: `DocumentUpload`, `ChatMessage`, `ExtractedDocument` from `app.db.models`
    - Import `get_db` from `app.db.session`, `get_engine` if available or use `engine` directly
    - Import `UserDTO` from `app.schemas.auth`
    - Router prefix: `/debug`, tags: `["debug"]`

    Endpoint 1 - POST /debug/restart-ingestion:
    - Depends on `require_tester`
    - Resets all DocumentUpload rows owned by the current user (current_user.id) where
      extraction_status is NOT "done": set extraction_status = "pending", module_source = None,
      extraction_method = None
    - Uses ORM session (db.query(DocumentUpload).filter(...).update({...})) then db.commit()
    - Returns `{"restarted": N}` where N is the number of rows updated

    Endpoint 2 - POST /debug/purge-coaching:
    - Depends on `require_tester`
    - Deletes all ChatMessage rows owned by the current user
    - Uses ORM: `db.query(ChatMessage).filter(ChatMessage.user_id == current_user.id).delete()`
      then db.commit()
    - Returns `{"deleted": N}` chat messages deleted

    Endpoint 3 - DELETE /debug/nuke:
    - Depends on `require_tester`
    - Deletes ALL data for the current user in this order (to respect FK constraints):
      1. ChatMessage (user_id = current_user.id)
      2. ExtractedDocument - join via DocumentUpload: delete where upload.user_id = current_user.id
      3. DocumentUpload (user_id = current_user.id)
    - For ExtractedDocument deletion use a subquery or explicit id list:
      ```python
      upload_ids = [u.id for u in db.query(DocumentUpload.id).filter(
          DocumentUpload.user_id == current_user.id).all()]
      if upload_ids:
          db.query(ExtractedDocument).filter(
              ExtractedDocument.upload_id.in_(upload_ids)).delete(synchronize_session=False)
      ```
    - db.commit() once after all deletes
    - Returns `{"nuked": True, "user_id": current_user.id}`

    IMPORTANT: All 3 endpoints are scoped to the current user only - never touch other users' data.

    - Add `app.include_router(debug_router)` after `app.include_router(notifications_router, ...)` line
