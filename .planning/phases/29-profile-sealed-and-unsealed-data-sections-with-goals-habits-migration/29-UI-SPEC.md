## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage                                                   |
| ----- | ----- | ------------------------------------------------------- |
| xs    | 4px   | Icon gaps, chip inline padding (`px-2.5 py-0.5`)        |
| sm    | 8px   | Gap between tags/chips, arrow-button outer gap          |
| md    | 16px  | Default element spacing, `space-y-4`                    |
| lg    | 24px  | Card padding `p-6`, section heading gap                 |
| xl    | 32px  | Layout vertical rhythm between sections                 |
| 2xl   | 48px  | Reserved standard token (not directly used in new components this phase) |
| 3xl   | 64px  | Not used in this phase                                                   |

Exceptions:
- Touch targets: `min-h-[44px]` (44px — WCAG 2.5.5 minimum; not a layout spacing token)
- Textarea minimum height: `min-h-[120px]` (3 lines at 16px × 1.5 lh + 2×16px padding)
- Carousel tab button fixed width: `min-w-[7rem] max-w-[9rem]` (112–144px; fixed so carousel translateX math is stable)
- Desktop scrollable strip bottom padding: `pb-1` (4px — prevents clipping of pill shadow)

## Color

Tokens are CSS custom properties defined in `src/index.css` — use semantic names only, never hex literals in component code.

| Role             | Light token         | Dark token          | Usage in this phase                                       |
| ---------------- | ------------------- | ------------------- | --------------------------------------------------------- |
| Dominant (60%)   | `--background`      | `--background`      | Page canvas, textarea background                          |
| Secondary (30%)  | `--card`            | `--card`            | Section card surfaces (`bg-card`)                         |
| Accent (10%)     | `--primary`         | `--primary`         | Active tab, chip background (`bg-primary/10`), save indicator |
| Muted surface    | `--muted`           | `--muted`           | Inactive tabs, disabled textarea, arrow buttons           |
| Muted text       | `--muted-foreground`| `--muted-foreground`| Labels, hint text, inactive tab labels                    |
| Destructive      | `--destructive`     | `--destructive`     | Error text, timeline badge dot                            |
| Border           | `--border`          | `--border`          | Card borders, input borders, `border-border`              |
| Foreground       | `--foreground`      | `--foreground`      | Primary text                                              |

Accent (`--primary`) is reserved for:
- Active/selected tab button background
- Goal/dos/donts chip background tint (`bg-primary/10 text-primary`)
- Save indicator text ("Salvato 🔒")
- Focus rings (`focus:ring-primary`)
- CTA button fills

**Sealed section visual differentiation:**
The "🔒 Solo per te" subsection uses the same card surface (`bg-card`) as "👁 Visibile all'AI". Visual distinction comes from the lock icon + distinct heading, not from a different background color — consistent with the app's monochromatic scheme.

**Locked state** (keyPair not available): sealed textarea gets `bg-muted text-muted-foreground cursor-not-allowed opacity-60`. Lock overlay is not a modal — it is an inline banner inside the sealed card.

## Component Inventory

### 1. ProfileScreen tab bar (updated)

#### Desktop — horizontally scrollable strip (D-12)

Replace current `hidden sm:flex mb-6 gap-2 flex-wrap` with:

```tsx
<div className="hidden sm:flex mb-6 gap-2 overflow-x-auto pb-1 [scrollbar-width:none] [&::-webkit-scrollbar]:hidden [-webkit-overflow-scrolling:touch]">
  {TABS.map(tab => (
    <button
      key={tab.id}
      onClick={() => handleTabClick(tab.id)}
      className={[
        "shrink-0 rounded-full px-4 py-1.5 text-sm font-medium transition-colors whitespace-nowrap",
        activeTab === tab.id
          ? "bg-primary text-primary-foreground"
          : "text-muted-foreground hover:text-foreground",
      ].join(" ")}
    >
      {tab.label}
    </button>
  ))}
</div>
```

Key change from current: add `overflow-x-auto`, `[scrollbar-width:none]`, `shrink-0` on each button, `whitespace-nowrap`. All 6 tabs remain in the DOM; user scrolls to reach "Preferenze".

#### Mobile — carousel with peeking adjacent tabs (D-13)

Replace current `sm:hidden grid grid-cols-2 gap-1` with:

```tsx
<div className="sm:hidden flex items-center gap-1 mb-4">
  {/* Prev arrow */}
  <button
    onClick={prevTab}
    disabled={activeTabIndex === 0}
    aria-label={t("profile.tabs.prev")}
    className="h-9 w-9 shrink-0 rounded-full bg-muted flex items-center justify-center transition-opacity disabled:opacity-30"
  >
    <ChevronLeft className="h-4 w-4 text-muted-foreground" />
  </button>

  {/* Clip container — overflow hidden creates peeking effect */}
  <div
    className="flex-1 overflow-hidden"
    onTouchStart={handleTouchStart}   // record e.touches[0].clientX
    onTouchEnd={handleTouchEnd}       // delta > 40px → prev/next
  >
    <div
      className="flex gap-2 transition-transform duration-200 ease-out"
      style={{ transform: `translateX(calc(${-activeTabIndex} * (var(--carousel-tab-w, 8rem) + 8px) + var(--carousel-peek-offset, 1.5rem)))` }}
    >
      {TABS.map((tab, i) => (
        <button
          key={tab.id}
          onClick={() => handleTabClick(tab.id)}
          className={[
            "shrink-0 rounded-lg px-3 py-2.5 text-sm font-medium min-h-[44px] transition-colors",
            "min-w-[7rem] max-w-[9rem] truncate",
            i === activeTabIndex
              ? "bg-primary text-primary-foreground"
              : "bg-muted text-muted-foreground",
          ].join(" ")}
        >
          {tab.label}
        </button>
      ))}
    </div>
  </div>

  {/* Next arrow */}
  <button
    onClick={nextTab}
    disabled={activeTabIndex === TABS.length - 1}
    aria-label={t("profile.tabs.next")}
    className="h-9 w-9 shrink-0 rounded-full bg-muted flex items-center justify-center transition-opacity disabled:opacity-30"
  >
    <ChevronRight className="h-4 w-4 text-muted-foreground" />
  </button>
</div>
```

Swipe implementation (raw touch — no library; mirror `usePullToRefresh.ts` pattern):
- `handleTouchStart`: store `touchStartX = e.touches[0].clientX`
- `handleTouchEnd`: if `touchStartX - e.changedTouches[0].clientX > 40` → nextTab(); if `< -40` → prevTab()
- Threshold: 40px (inherited from typical mobile swipe threshold)

CSS variable `--carousel-peek-offset: 1.5rem` offsets the track so the previous tab peeks on the left when `activeTabIndex > 0`. This variable can be removed and replaced with a computed offset if simpler.

### 4. `SealedNotesSection` (new component)

File: `senso/src/features/profile/SealedNotesSection.tsx` (or inline in `PreferenzaTab.tsx`)

#### Normal state (keyPair available)

```tsx
<section className="rounded-2xl border border-border bg-card p-6 space-y-4">
  <div className="flex items-center gap-2">
    <Lock className="h-4 w-4 text-muted-foreground" />
    <h2 className="text-base font-semibold">{t("profile.preferences.sealedHeading")}</h2>
  </div>
  <p className="text-xs text-muted-foreground">{t("profile.preferences.sealedHint")}</p>

  <div className="relative">
    <textarea
      value={plaintext}
      onChange={e => setPlaintext(e.target.value)}
      onBlur={handleSealedBlur}  // encrypt → PATCH on blur
      placeholder={t("profile.preferences.sealedPlaceholder")}
      className="w-full rounded-lg border border-border bg-background px-3 py-2 text-sm text-foreground
                 placeholder:text-muted-foreground resize-none min-h-[120px]
                 focus:outline-none focus:ring-2 focus:ring-primary"
    />
    {/* Inline save indicator — shown briefly after successful PATCH */}
    {saveState === "saved" && (
      <span className="absolute bottom-2 right-2 text-xs text-primary animate-fade-out">
        {t("profile.preferences.savedIndicator")}
      </span>
    )}
    {saveState === "error" && (
      <span className="absolute bottom-2 right-2 text-xs text-destructive">
        {t("profile.preferences.saveError")}
      </span>
    )}
  </div>
</section>
```

`handleSealedBlur`: encrypt current `plaintext` with `keyPair.publicKey` via `nacl.crypto_box_seal` → `PATCH /api/profile/sealed-notes` → set `saveState = "saved"` → clear after 2s.

## Interaction States

### Auto-save lifecycle (sealed notes)

```
User types → onBlur fires
  → encrypt(plaintext, publicKey)
  → PATCH /api/profile/sealed-notes {ciphertext: base64}
  → [success] saveState = "saved" → show "Salvato 🔒" → clear after 2000ms
  → [network error] saveState = "error" → show error inline → stays until next blur
```

No loading spinner during save (fire-and-forget on blur; next blur resets state).

### Tab carousel swipe gesture

```
touchStart: store touchStartX
touchEnd:
  delta = touchStartX - changedTouches[0].clientX
  if delta > 40 && activeTabIndex < maxIndex → nextTab()
  if delta < -40 && activeTabIndex > 0 → prevTab()
```

### Carousel arrow button states

| Condition                     | Style                                   |
| ----------------------------- | --------------------------------------- |
| Enabled                       | `bg-muted text-muted-foreground`        |
| Disabled (first/last tab)     | `opacity-30 cursor-not-allowed`         |

### TagInput (unchanged from existing SettingsScreen implementation)

Desktop: Enter key or comma adds chip. Mobile: comma auto-adds; "Aggiungi" button is the primary path (shown only on `useIsMobile()`).
