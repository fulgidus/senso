# Plan 22-04 — Profile Tabs + Admin Route + Nav Tap Targets

## Objective

Fix the profile tab bar which overflows on 320px screens (horizontal scroll of many tabs).
Replace with 2×N grid layout on mobile. Move admin/debug to dedicated route. Ensure all
nav items have minimum 44px tap targets.

---

<task id="22-04-01" title="Profile tab bar: grid layout on mobile">
  <objective>
    Replace the horizontal overflow tab strip in ProfileScreen with a 2-column grid of
    tab buttons on `< sm` breakpoints.
  </objective>
  <read_first>
    - senso/src/features/profile/ProfileScreen.tsx — tab bar implementation, tab names,
      active tab state
    - senso/src/locales/it.json — tab label keys
  </read_first>
  <action>
    Replace the tab bar markup:

    ```tsx
    {/* Mobile: 2-column grid */}
    <div className="sm:hidden grid grid-cols-2 gap-1 mb-4">
      {tabs.map(tab => (
        <button
          key={tab.id}
          onClick={() => setActiveTab(tab.id)}
          className={cn(
            "py-2 px-3 rounded-lg text-sm font-medium transition-colors min-h-[44px]",
            activeTab === tab.id
              ? "bg-primary text-primary-foreground"
              : "bg-muted text-muted-foreground hover:bg-muted/80"
          )}
        >
          {tab.label}
        </button>
      ))}
    </div>

    {/* Desktop: horizontal tab strip (existing) */}
    <div className="hidden sm:flex border-b gap-1 mb-4 overflow-x-auto">
      {tabs.map(tab => (/* existing desktop tab buttons */))}
    </div>
    ```

    Ensure each grid tab button has `min-h-[44px]` (44px tap target requirement).
  </action>
  <acceptance_criteria>
    - At 320px width: tabs render in 2-column grid, no horizontal overflow
    - At ≥ 640px: original horizontal tab strip unchanged
    - Each tab button has ≥ 44px height
    - Active tab visually distinct in both layouts
    - `pnpm build` succeeds
  </acceptance_criteria>
</task>
