# Plan 22-03 — All Tables → Card Layouts on Mobile (Systemic)

## Objective

Audit all 6 tables in the app and add parallel card list views for `< sm` breakpoints.
Pattern: `<table className="hidden sm:table">` + `<ul className="sm:hidden space-y-2">`
with same data in stacked label+value format. Sort controls above card list on mobile.

---

  <objective>
    Find all 6 table components (admin content, files tab, transactions, admin users,
    notifications, connectors placeholder), add mobile card layout to each.
  </objective>
  <read_first>
    - senso/src/features/admin/AdminContentPage.tsx
    - senso/src/features/profile/FilesTab.tsx
    - senso/src/features/profile/TransactionsSection.tsx (or equivalent)

  <action>
    For EACH table, apply this pattern:

    ```tsx
    {/* Desktop: table */}
    <div className="hidden sm:block">
      <table className="w-full text-sm">
        <thead>...</thead>
        <tbody>
          {items.map(item => (
            <tr key={item.id}>
              <td>{item.field1}</td>
              <td>{item.field2}</td>
              {/* ... */}
            </tr>
          ))}
        </tbody>
      </table>
    </div>

    {/* Mobile: card list */}
    <ul className="sm:hidden space-y-2">
      {items.map(item => (
        <li key={item.id} className="rounded-lg border p-3 text-sm space-y-1">
          <div className="flex justify-between">
            <span className="text-muted-foreground">{t('col.field1')}</span>
            <span className="font-medium">{item.field1}</span>
          </div>
          <div className="flex justify-between">
            <span className="text-muted-foreground">{t('col.field2')}</span>
            <span>{item.field2}</span>
          </div>
          {/* Primary action button if applicable */}
        </li>
      ))}
    </ul>
    ```

    If the table has sortable columns, add a sort control above the card list:
    ```tsx
    <div className="sm:hidden mb-2">
      <select
        value={sortKey}
        onChange={e => setSortKey(e.target.value)}
        className="w-full rounded border px-2 py-1 text-sm"
      >
        {columns.map(c => <option key={c.key} value={c.key}>{c.label}</option>)}
      </select>
    </div>
    ```

    4. Admin users table (AdminUsersPage) — columns: username, email, role, created_at
    5. Notifications panel (NotificationsPanel) — currently a list but ensure no horizontal overflow
    6. Connectors tab (ConnectorsTab) — placeholder table/grid if present

    Confirm all 6 are covered. If a component doesn't use `<table>`, check for horizontal
    overflow issues and fix (e.g., `overflow-x-auto` wrapper, or truncate long text with
