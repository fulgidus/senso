# Phase 22 - UI Design Contract

> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> initialized with an established palette. All decisions below document *what already
> exists* PLUS the new mobile-specific patterns introduced in this phase.

## Spacing Scale

Declared values (4px base grid вҖ” all Tailwind utilities already in use):

| Token | Value | Usage                                              |
| ----- | ----- | -------------------------------------------------- |
| xs    | 4px   | Icon gaps, badge padding, inline spacing           |
| sm    | 8px   | Compact element spacing, gap-2 in card rows        |
| md    | 16px  | Default element spacing, card padding (p-4)        |
| lg    | 24px  | Section padding, major card spacing                |
| xl    | 32px  | Layout gaps, page-level section breaks             |
| 2xl   | 48px  | Major section breaks                               |
| 3xl   | 64px  | Page-level spacing                                 |

**Mobile-specific additions:**
- Card row height minimum: `min-h-[44px]` вҖ” all tap targets must meet 44px (iOS HIG)
- Mobile card item padding: `p-3` (12px) вҖ” one step below desktop `p-4`
- PTR indicator height: `h-12` (48px) вҖ” visible above message list, not intrusive
- Scroll-to-bottom FAB: `size-10` (40px) + safe-area offset from bottom

Exceptions: none вҖ” all values are multiples of 4.

## Typography

| Role    | Size  | Weight   | Line Height | Notes                       |
| ------- | ----- | -------- | ----------- | --------------------------- |
| Body    | 16px  | 400      | 1.5         | App default, `text-base`    |
| Label   | 14px  | 500      | 1.4         | `text-sm font-medium`       |
| Caption | 12px  | 400      | 1.4         | `text-xs text-muted-foreground` |
| Heading | 20px  | 600      | 1.3         | `text-xl font-semibold`     |
| Display | 24px  | 700      | 1.2         | `text-2xl font-bold`        |

**Mobile card typography rules:**
- Table column headers become caption labels (`text-xs text-muted-foreground`) left of values
- Values use body weight (`text-sm font-medium`) right-aligned or left-aligned per context
- Sort control label: `text-sm font-medium` (matches label)
- PTR "Aggiornamento..." text: `text-sm text-muted-foreground` centered

All strings must come from `senso/src/locales/it.json` and `en.json` вҖ” no hardcoded
Italian strings in component files.

### Accent Reserved For
- Active tab indicators
- Focus rings on inputs
- Primary action buttons
- PTR indicator spinner border
- Scroll-to-bottom button background on hover

Accent is **NOT** used for: general card borders, table row hovers, informational text,
coaching message bubbles, muted labels.

### Phase 22 New Color Rules

**Coach picker dark mode fix (22-05):**
- Coach picker modal card container must use `bg-card` (not `bg-background`) in both
  modes. In dark mode `bg-card = #1A3A5C`, not `#112D4E` вҖ” this is the fix for the
  unreadable light background issue.
- Coach avatar ring: `ring-primary` (2px), transparent gap on card bg.
- Selected coach highlight: `bg-primary/10 border-primary` in both modes.

**PTR indicator (22-02):**
- Spinner: `border-primary` (current color) + `border-t-transparent` (spin animation)
- Background: none (transparent over message list, semi-transparent if needed: `bg-background/80`)
- "Aggiornamento..." text: `text-muted-foreground`

**Scroll-to-bottom FAB (22-01):**
- Background: `bg-primary` or `bg-card border border-border` (prefer bordered variant to avoid
  covering content with solid fill)
- Icon: `text-primary-foreground` or `text-foreground`
- Hover: `hover:bg-primary/90` or `hover:bg-muted`
- Position: `absolute bottom-4 right-4` above keyboard safe area

**Mobile table card items (22-03):**
- Card: `rounded-lg border border-border bg-card p-3`
- Label column: `text-muted-foreground`
- Value column: `text-foreground font-medium`
- Action button within card: existing shadcn `Button` variant `ghost` or `outline`

**Profile tab grid buttons (22-04):**
- Active: `bg-primary text-primary-foreground`
- Inactive: `bg-muted text-muted-foreground hover:bg-muted/80`
- Matches existing horizontal tab strip token usage вҖ” just re-implemented as grid

## New Component Patterns (Phase 22 only)

### 1. Scroll-to-Bottom FAB

```tsx
// Appearance: circular button, 40px, bottom-right of chat container
// Visibility: conditional on showScrollBtn state
// Position: absolute within chat scroll container (not fixed вҖ” avoids iOS keyboard issues)
<button
  aria-label={t('chat.scrollToBottom.label')}
  className={cn(
    "absolute bottom-4 right-4 z-10 size-10 rounded-full",
    "flex items-center justify-center",
    "bg-card border border-border shadow-sm",
    "hover:bg-muted transition-colors",
    "text-foreground",
    !showScrollBtn && "hidden"
  )}
  onClick={scrollToBottom}
>
  <ChevronDown className="size-4" />
</button>
```

### 2. Pull-to-Refresh Indicator

```tsx
// Appears above message list during refresh
// Height: h-12 (48px), centered spinner + text
<div
  className={cn(
    "flex items-center justify-center gap-2 h-12",
    "text-sm text-muted-foreground",
    !refreshing && "hidden"
  )}
  aria-live="polite"
  aria-label={t('chat.pullToRefresh.loading')}
>
  <div className="size-4 rounded-full border-2 border-primary border-t-transparent animate-spin" />
  <span>{t('chat.pullToRefresh.loading')}</span>
</div>
```

### 3. Mobile Table Card Item

```tsx
// Pattern: used for each of the 6 tables in phase 22-03
// Container: <ul className="sm:hidden space-y-2">
<li className="rounded-lg border border-border bg-card p-3 text-sm space-y-1">
  <div className="flex justify-between items-start">
    <span className="text-muted-foreground">{t('col.fieldName')}</span>
    <span className="font-medium text-right">{item.value}</span>
  </div>
  {/* Repeat for each column */}
  {/* Action button row at bottom if applicable */}
  <div className="flex gap-2 pt-1">
    {/* <Button variant="outline" size="sm"> ... </Button> */}
  </div>
</li>
```

### 4. Mobile Sort Control

```tsx
// Appears above card list on mobile when table is sortable
// Hidden at sm+ breakpoints
<div className="sm:hidden mb-2 flex items-center gap-2">
  <label className="text-sm font-medium shrink-0">{t('table.sort.label')}</label>
  <select
    className="flex-1 rounded-md border border-border bg-background px-3 py-1.5 text-sm"
    value={sortKey}
    onChange={e => setSortKey(e.target.value)}
  >
    {columns.map(col => (
      <option key={col.key} value={col.key}>{col.label}</option>
    ))}
  </select>
</div>
```

### 5. Profile Tab Grid (Mobile)

```tsx
// Mobile: 2-column grid of buttons replacing horizontal overflow strip
// Renders at < sm breakpoints only
<div className="sm:hidden grid grid-cols-2 gap-1 mb-4">
  {tabs.map(tab => (
    <button
      key={tab.id}
      onClick={() => setActiveTab(tab.id)}
      className={cn(
        "py-2 px-3 rounded-lg text-sm font-medium transition-colors min-h-[44px]",
        activeTab === tab.id
          ? "bg-primary text-primary-foreground"
          : "bg-muted text-muted-foreground hover:bg-muted/80"
      )}
    >
      {t(tab.labelKey)}
    </button>
  ))}
</div>
```

## PWA Manifest Contract (22-05)

Not a UI component spec, but locked here to avoid ad-hoc changes:

```json
{
  "name": "S.E.N.S.O.",
  "short_name": "SENSO",
  "display": "standalone",
  "start_url": "/",
  "background_color": "#ffffff",
  "theme_color": "#3f72af",
  "icons": [
    { "src": "/pwa-192x192.png", "sizes": "192x192", "type": "image/png", "purpose": "any" },
    { "src": "/pwa-512x512.png", "sizes": "512x512", "type": "image/png", "purpose": "any" },
    { "src": "/maskable-icon-512x512.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable" }
  ]
}
```

**Rules:**
- NEVER combine `purpose: 'any maskable'` in one entry вҖ” Chrome/Android rejects it
- `theme_color` matches `--primary` light mode value (`#3F72AF`)
- `background_color` matches `--background` light mode value (`#FFFFFF`)
