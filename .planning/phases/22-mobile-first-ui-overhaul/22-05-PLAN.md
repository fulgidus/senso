# Plan 22-05 - PWA Standalone + Coach Picker Dark Mode + Session Fix

## Objective

Fix PWA manifest so the app opens in standalone mode (no browser address bar). Fix coach
picker modal which renders with an unreadable light background in dark mode. Fix the bug
where switching to a non-default coach breaks the session (persona_id not persisted).

<task id="22-05-02" title="Coach picker dark mode fix + session persistence fix">
  <objective>
    Trace and fix the coach picker modal's light background in dark mode. Trace and fix
    the persona_id not persisting when switching coaches (non-default coach breaks session).
  </objective>
  <read_first>
    - senso/src/features/coaching/ChatScreen.tsx - CoachPickerModal usage, persona state
    - senso/src/features/coaching/CoachPickerModal.tsx (or equivalent) - modal markup
    - api/app/coaching/service.py - how persona_id is used in chat(), session creation
    - api/app/db/models.py - CoachingSession model, persona_id field
  </read_first>
  <action>
    **Dark mode fix:**
    In CoachPickerModal (or the coach card component), trace where the white/light
    background comes from. Common causes:
    - Hardcoded `bg-white` or `bg-gray-50` - replace with `bg-background`
    - Dialog/Sheet content without `dark:bg-background` - add explicit dark: override
    - Inline `style={{ background: '#fff' }}` - replace with Tailwind token

    Add `dark:bg-card` or `dark:bg-popover` to the coach card container explicitly.

    **Session persistence fix:**
    Trace the persona_id flow:
    1. User selects coach in picker → `persona_id` stored in localStorage/state
    2. New chat message sent → `persona_id` included in request body?
    3. `CoachingService.chat()` → reads `persona_id` from request → creates/uses session
    4. Check: does `CoachingSession` store `persona_id`? Is it read back on subsequent
       messages?

    Likely fix: ensure `POST /coaching/chat` sends `persona_id` on every message (not
    just the first), and that `get_or_create_session()` uses the provided `persona_id`
    not a cached/default one.

    If persona_id is not included in the chat request DTO, add it and wire through.
  </action>
  <acceptance_criteria>
    - Coach picker card backgrounds use semantic tokens (dark:bg-card) - no hardcoded colors
    - Switch to non-default coach → subsequent messages use that coach's persona
    - Dark mode: coach picker cards readable (dark background in dark mode)
    - Switching back to default coach works correctly
  </acceptance_criteria>
</task>
