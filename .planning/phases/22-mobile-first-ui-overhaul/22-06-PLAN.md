# Plan 22-06 - TTS Fix + Voice UX + STT Chromium

## Objective

Fix the TTS (text-to-speech) broken path: ElevenLabs audio blob not playing due to
`canplay` event not firing or blob URL revoked before play. Add manual play button
as fallback for autoplay policy. Fix STT microphone ref contention if still broken.

<task id="22-06-01" title="Trace + fix TTS audio playback pipeline">
  <objective>
    Diagnose and fix why ElevenLabs TTS audio fails to play in the browser. The bug
    is in the audio blob → `<audio>` element → play() pipeline.
  </objective>
  <read_first>
    - senso/src/features/coaching/ChatScreen.tsx - TTS triggering, audio element, blob URL
    - senso/src/hooks/useTTS.ts (if exists) - TTS hook
    - api/app/coaching/tts.py - ElevenLabs TTS endpoint, response format
  </read_first>
  <action>
    Trace the full pipeline:
    1. `POST /coaching/tts` → ElevenLabs API → returns audio bytes
    2. Frontend: `fetch → arrayBuffer → new Blob → URL.createObjectURL`
    3. `new Audio(blobURL)` → `.play()`

    Common failure modes to check and fix:
    - **Blob URL revoked too early**: Only `URL.revokeObjectURL(url)` AFTER audio ends,
      not before play: `audio.addEventListener('ended', () => URL.revokeObjectURL(url))`
    - **autoplay blocked**: Browser blocks `audio.play()` without user gesture.
      Fix: catch the rejected Promise and show a manual play button:
      ```tsx
      audio.play().catch((err) => {
        if (err.name === 'NotAllowedError') {
          setShowPlayButton(true); // render manual play button
        }
      });
      ```
    - **`canplay` not firing**: Use `audio.play()` Promise directly instead of waiting
      for `canplay` event. The Promise resolves when playback actually starts.
    - **Content-Type**: Ensure TTS endpoint returns `audio/mpeg` content-type header.

    **Fallback**: If ElevenLabs is unavailable (503), fall back to `speechSynthesis.speak()`:
    ```tsx
    const fallback = new SpeechSynthesisUtterance(text);
    fallback.lang = 'it-IT';
    fallback.rate = 1.0;
    window.speechSynthesis.speak(fallback);
    ```

    In `tts.py`: verify ElevenLabs SDK call returns binary audio; add try/except and
    return 503 with `{"error": "tts_unavailable"}` if ElevenLabs fails.
  </action>
  <acceptance_criteria>
    - TTS plays audio after coach response (manual test with ElevenLabs key in env)
    - Blob URL revoked AFTER audio ends, not before play
    - `NotAllowedError` caught and manual play button shown (not silent failure)
    - 503 from ElevenLabs → speechSynthesis fallback (not spinner forever)
    - `docker compose run --rm api /app/.venv/bin/python -c "import app; print('ok')"` passes
  </acceptance_criteria>
</task>
