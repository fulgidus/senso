# Plan 22-01 - ChatScreen: visualViewport Keyboard + Safe Area + Scroll Button

## Objective

Fix the chat input disappearing under the iOS software keyboard. The root cause: iOS does
NOT fire `window.resize` when the keyboard opens - `visualViewport.resize` is required.
Also add a scroll-to-bottom button and fix `overscroll-behavior` for iOS.

See 22-RESEARCH.md for verified iOS keyboard patterns - use those exactly.

---

<task id="22-01-01" title="useKeyboardHeight hook via visualViewport">
  <objective>
    Create `senso/src/hooks/useKeyboardHeight.ts` that returns the current keyboard
    height in pixels using `window.visualViewport`. This is the only API that works
    reliably on iOS Safari.
  </objective>
  <read_first>
    - senso/src/hooks/ - existing hook patterns
    - 22-RESEARCH.md - exact visualViewport pattern to use
  </read_first>
  <action>
    ```tsx
    // senso/src/hooks/useKeyboardHeight.ts
    import { useState, useEffect } from 'react';

    export function useKeyboardHeight(): number {
      const [keyboardHeight, setKeyboardHeight] = useState(0);

      useEffect(() => {
        const vv = window.visualViewport;
        if (!vv) return;

        const update = () => {
          // keyboard height = total window height - visible viewport - any scroll offset
          const kh = Math.max(0, window.innerHeight - vv.height - vv.offsetTop);
          setKeyboardHeight(kh);
        };

        vv.addEventListener('resize', update);
        vv.addEventListener('scroll', update); // iOS also fires scroll on vv
        return () => {
          vv.removeEventListener('resize', update);
          vv.removeEventListener('scroll', update);
        };
      }, []);

      return keyboardHeight;
    }
    ```

    Note: The hook intentionally returns `0` when `visualViewport` is unavailable
    (e.g., older browsers) - graceful degradation.
  </action>
  <acceptance_criteria>
    - Hook file exists and exports `useKeyboardHeight`
    - TypeScript compiles without errors (`pnpm build` succeeds)
    - Returns 0 when no keyboard is open
  </acceptance_criteria>
</task>

<task id="22-01-02" title="ChatScreen input anchored above keyboard + scroll button">
  <objective>
    Apply `useKeyboardHeight` to the chat input container so it rises above the iOS
    keyboard. Fix `position: fixed` → `position: sticky bottom-0`. Add scroll-to-bottom
    button that appears when user is not at the end of the message list.
  </objective>
  <read_first>
    - senso/src/features/coaching/ChatScreen.tsx - input layout, listRef, message list
    - senso/src/locales/it.json - for any new i18n strings
  </read_first>
  <action>
    1. Replace any `position: fixed` or `position: absolute bottom-0` on the input
       container with `position: sticky; bottom: 0`.

    2. Apply keyboard height as CSS variable:
       ```tsx
       const keyboardHeight = useKeyboardHeight();
       // On the input wrapper:
       style={{ paddingBottom: `max(env(safe-area-inset-bottom, 0px), ${keyboardHeight}px)` }}
       ```

    3. Ensure the outer chat container is `display: flex; flex-direction: column;
       height: 100%` so `sticky` works correctly.

    4. Set `overscroll-behavior: contain` on the message list (replaces `overscroll-none`
       which breaks iOS momentum scroll).

    5. Add scroll-to-bottom button:
       ```tsx
       const [showScrollBtn, setShowScrollBtn] = useState(false);
       // Track scroll position in listRef
       const onScroll = () => {
         const el = listRef.current;
         if (!el) return;
         setShowScrollBtn(el.scrollHeight - el.scrollTop - el.clientHeight > 100);
       };
       // Render: {showScrollBtn && <button onClick={scrollToBottom}>↓</button>}
       ```

    6. Ensure `<meta name="viewport">` in `senso/index.html` includes `viewport-fit=cover`.
  </action>
  <acceptance_criteria>
    - Chat input visible on iOS when keyboard opens (manual test)
    - Input uses `sticky bottom-0` not `fixed`
    - Scroll-to-bottom button appears when scrolled up > 100px from bottom
    - `pnpm build` succeeds
    - `viewport-fit=cover` present in index.html meta viewport
  </acceptance_criteria>
</task>
