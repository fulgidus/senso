# Plan 25-01: useIsMobile hook + ChatScreen + session rename Enter-submit removal

## Goal

Add a shared `useIsMobile()` hook based on the existing `useMediaQuery` infrastructure, then apply it to two inputs in `ChatScreen.tsx`:
1. **Chat textarea** ÔÇö on mobile, Enter inserts a newline instead of submitting; send button is the sole submit path.
2. **Session rename input** ÔÇö on mobile, Enter no longer commits the rename; user must blur or press another key.

Desktop behavior stays identical (Enter submits/commits as before).

### Task 1 ÔÇö Create useIsMobile hook

<read_first>
- `senso/src/hooks/useMediaQuery.ts` ÔÇö understand the existing hook pattern (useSyncExternalStore)
- `senso/src/hooks/useReducedMotion.ts` ÔÇö example of a simple hook that wraps useMediaQuery
</read_first>

<action>
Create `senso/src/hooks/useIsMobile.ts`:

```typescript
import { useMediaQuery } from "./useMediaQuery"

/**
 * Returns true when the primary pointer device is coarse (touch).
 * Uses `(pointer: coarse)` media query ÔÇö reliable across iOS, Android, and desktop.
 * Desktop mice return false; mobile touchscreens return true.
 *
 * Server-side snapshot defaults to false (desktop fallback, consistent with useMediaQuery).
 */
export function useIsMobile(): boolean {
  return useMediaQuery("(pointer: coarse)")
}
```
</action>

```typescript
const handleKeyDown = (e: React.KeyboardEvent<HTMLTextAreaElement>) => {
```

```typescript
    // On desktop: Enter submits (Shift+Enter still inserts newline ÔÇö existing behavior, unchanged).
    if (!isMobile && e.key === "Enter" && !e.shiftKey) {
        e.preventDefault()
        void handleSend()
    }
}
```
