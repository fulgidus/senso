**Step C: Add chip helper function inside TagInput**

```typescript
const addChip = () => {
    const trimmed = input.trim()
    if (trimmed && !items.includes(trimmed)) {
        onChange([...items, trimmed])
    }
    setInput("")
}
```

**Step D: Update the `onKeyDown` handler**

**Before:**
```tsx
onKeyDown={(e) => {
    if (e.key === "Enter" && input.trim()) {
        e.preventDefault()
        if (!items.includes(input.trim())) {
            onChange([...items, input.trim()])
        }
        setInput("")
    }
}}
```

**After:**
```tsx
onKeyDown={(e) => {
    // Desktop: Enter OR comma adds a chip (D-06)
    // Mobile: comma auto-adds (D-07); Enter does nothing (Add button is the only path)
    if (e.key === "," || (!isMobile && e.key === "Enter")) {
        e.preventDefault()
        addChip()
    }
}}
```

**Note on comma behavior:** When the user types `,`, `e.preventDefault()` prevents the comma itself from appearing in the input. `addChip()` is called immediately (works for both desktop and mobile).

**Step E: Add mobile "Add" button**

Render the button below the input, only on mobile:
```tsx
{isMobile && (
    <button
        type="button"
        onClick={addChip}
        disabled={!input.trim()}
        className="mt-1.5 w-full rounded-md border border-input bg-background px-3 py-1.5 text-sm text-primary hover:bg-primary/5 disabled:opacity-40 disabled:cursor-not-allowed"
    >
        {addButtonLabel}
    </button>
)}
```

**Step F: Update TagInput call sites in SettingsScreen**

The three `<TagInput>` renders (goals, dos, donts) need the new prop. Add `addButtonLabel={t("preferences.addButton")}` to each:

```tsx

<TagInput items={dos}   onChange={...} placeholder={t("preferences.dosPlaceholder")}   addButtonLabel={t("preferences.addButton")} />
<TagInput items={donts} onChange={...} placeholder={t("preferences.dontsPlaceholder")} addButtonLabel={t("preferences.addButton")} />
```
</action>

## Risks

- `e.key === ","` fires even if the user holds Shift (which produces `<` on some keyboards). If comma-add is undesirable with Shift, add `&& !e.shiftKey` guard. For this use case (tags), comma with Shift is fine.
- The Add button `disabled` state requires `input.trim()` to be truthy — this prevents empty chip submission on mobile where comma auto-submits.
- The `addHint` text change affects desktop hint too. On mobile the hint could say "Use comma or the Add button" but changing to "virgola o pulsante" is out of scope; the updated text stays generic.
