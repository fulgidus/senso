# Plan: Demo Seed Script + Reset Script

## Goal

Provide two one-command scripts - `seed-demo.sh` creates a demo user account and uploads the sample financial files via the API; `reset-demo.sh` wipes that user's data so the demo can be re-run from a clean slate.

## Context

**Available sample files** (gitignored, never shipped in Docker image):
- `api/app/ingestion/samples/revolut_it/RevolutIT_account-statement_2026-02-01_2026-03-24_en-us_4700b8.csv`
- `api/app/ingestion/samples/revolut_it/RevolutIT_account-statement_2026-02-01_2026-03-24_en-us_d8fee2.pdf`
- `api/app/ingestion/samples/fineco_it/FinecoIT_movements_20260324.xlsx`
- `api/app/ingestion/samples/satispay_it/SatispayIT_Export_Report.xlsx`
- `api/app/ingestion/samples/satispay_it/SatispayIT_Export_Report.pdf`

**Existing API endpoints used by the script:**
- `POST /auth/signup` - `{"email": "...", "password": "..."}`  → `{"access_token": "...", "refresh_token": "..."}`
- `POST /auth/login` - same shape → tokens
- `POST /ingestion/upload` - multipart file + `Authorization: Bearer <token>` → `{"upload_id": "..."}`
- `GET /ingestion/uploads` - list all uploads for the user
- `GET /ingestion/uploads/{upload_id}` - poll status (`status: "done" | "extracting" | "error"`)
- `POST /ingestion/confirm-all` - confirm all completed uploads + triggers categorization
- `GET /profile/status` - poll categorization status (`status: "done" | "in_progress"`)

**Admin endpoint for reset:**
- `DELETE /auth/users/{user_id}` - admin-only, requires `is_admin=True` on the admin account. Alternatively, use a direct DB call since we are already inside Docker.

**Design decision** (from CONTEXT.md open question 1): Use the existing API endpoints (register → upload → confirm). This is more realistic and tests the actual demo path. Direct DB writes are not used.

**Design decision** (from CONTEXT.md open question 4): Full reset - clear uploads, chat sessions, and profile so the demo restarts from the upload step. This gives the cleanest re-demo experience.

## Tasks

### Task 1: Create `scripts/seed-demo.sh`

**Files to change:**
- `scripts/seed-demo.sh` - new file, executable bash script

**Steps:**

1. **Create `scripts/seed-demo.sh`** with the following structure:

```bash
#!/usr/bin/env bash
# seed-demo.sh - Create demo user + upload sample financial files via API
# Usage: bash scripts/seed-demo.sh
# Requirements: curl, Docker Compose stack running (docker compose up -d)

set -euo pipefail

API_URL="${API_URL:-http://localhost:8000}"
DEMO_EMAIL="${DEMO_EMAIL:-demo@senso.app}"
DEMO_PASSWORD="${DEMO_PASSWORD:-SensoDEMO2026!}"
SAMPLES_DIR="$(dirname "$0")/../api/app/ingestion/samples"

# ── Colors ───────────────────────────────────────────────────────────────────
GREEN='\033[0;32m'; YELLOW='\033[1;33m'; RED='\033[0;31m'; NC='\033[0m'
ok()   { echo -e "${GREEN}✓${NC} $*"; }
warn() { echo -e "${YELLOW}⚠${NC}  $*"; }
fail() { echo -e "${RED}✗${NC} $*"; exit 1; }

echo ""
echo "╔════════════════════════════════════════╗"
echo "║   S.E.N.S.O. Demo Seed Script          ║"
echo "╚════════════════════════════════════════╝"
echo ""

# ── Step 1: Wait for API ──────────────────────────────────────────────────────
echo "Waiting for API at $API_URL ..."
for i in $(seq 1 20); do
  if curl -sf "$API_URL/health" > /dev/null 2>&1; then
    ok "API is up"
    break
  fi
  if [ "$i" -eq 20 ]; then
    fail "API did not respond after 20 attempts. Run: docker compose up -d"
  fi
  sleep 2
done

# ── Step 2: Create or login demo user ────────────────────────────────────────
echo ""
echo "Creating demo user: $DEMO_EMAIL ..."
SIGNUP_RESP=$(curl -sf -X POST "$API_URL/auth/signup" \
  -H "Content-Type: application/json" \
  -d "{\"email\": \"$DEMO_EMAIL\", \"password\": \"$DEMO_PASSWORD\"}" 2>&1) || true

if echo "$SIGNUP_RESP" | grep -q "access_token"; then
  ok "Demo user created"
  ACCESS_TOKEN=$(echo "$SIGNUP_RESP" | python3 -c "import sys,json; print(json.load(sys.stdin)['tokens']['access_token'])")
else
  warn "Signup failed (user may already exist) - trying login..."
  LOGIN_RESP=$(curl -sf -X POST "$API_URL/auth/login" \
    -H "Content-Type: application/json" \
    -d "{\"email\": \"$DEMO_EMAIL\", \"password\": \"$DEMO_PASSWORD\"}" 2>&1) \
    || fail "Login failed. Run reset-demo.sh first if the account state is inconsistent."
  ACCESS_TOKEN=$(echo "$LOGIN_RESP" | python3 -c "import sys,json; print(json.load(sys.stdin)['tokens']['access_token'])")
  ok "Logged in as existing demo user"
fi

AUTH_HEADER="Authorization: Bearer $ACCESS_TOKEN"

# ── Step 3: Upload sample files ────────────────────────────────────────────
echo ""
echo "Uploading sample files from $SAMPLES_DIR ..."

UPLOAD_IDS=()

upload_file() {
  local filepath="$1"
  local filename
  filename=$(basename "$filepath")
  if [ ! -f "$filepath" ]; then
    warn "Sample file not found, skipping: $filepath"
    return
  fi
  echo -n "  Uploading $filename ... "
  RESP=$(curl -sf -X POST "$API_URL/ingestion/upload" \
    -H "$AUTH_HEADER" \
    -F "file=@${filepath}" 2>&1) || { echo "FAILED"; warn "Upload failed for $filename"; return; }
  UPLOAD_ID=$(echo "$RESP" | python3 -c "import sys,json; print(json.load(sys.stdin).get('upload_id',''))" 2>/dev/null || echo "")
  if [ -n "$UPLOAD_ID" ]; then
    UPLOAD_IDS+=("$UPLOAD_ID")
    echo "OK (id: $UPLOAD_ID)"
  else
    echo "FAILED"
    warn "No upload_id in response: $RESP"
  fi
}

# Upload the core demo files - Revolut CSV is the primary data source
upload_file "$SAMPLES_DIR/revolut_it/RevolutIT_account-statement_2026-02-01_2026-03-24_en-us_4700b8.csv"
upload_file "$SAMPLES_DIR/fineco_it/FinecoIT_movements_20260324.xlsx"
upload_file "$SAMPLES_DIR/satispay_it/SatispayIT_Export_Report.xlsx"

if [ ${#UPLOAD_IDS[@]} -eq 0 ]; then
  fail "No files were uploaded. Check that sample files exist at $SAMPLES_DIR"
fi

ok "Uploaded ${#UPLOAD_IDS[@]} file(s)"

# ── Step 4: Wait for extraction to complete ───────────────────────────────────
echo ""
echo "Waiting for extraction to complete (up to 60s per file) ..."
for UPLOAD_ID in "${UPLOAD_IDS[@]}"; do
  echo -n "  Waiting for upload $UPLOAD_ID ... "
  for i in $(seq 1 30); do
    STATUS_RESP=$(curl -sf "$API_URL/ingestion/uploads/$UPLOAD_ID" \
      -H "$AUTH_HEADER" 2>&1) || { echo "POLL_ERROR"; break; }
    STATUS=$(echo "$STATUS_RESP" | python3 -c "import sys,json; print(json.load(sys.stdin).get('status','unknown'))" 2>/dev/null || echo "unknown")
    if [ "$STATUS" = "done" ] || [ "$STATUS" = "confirmed" ]; then
      echo "done"
      break
    elif [ "$STATUS" = "error" ]; then
      echo "error (continuing)"
      break
    fi
    if [ "$i" -eq 30 ]; then
      echo "timeout (status=$STATUS, continuing)"
    else
      sleep 2
    fi
  done
done

# ── Step 5: Confirm all + trigger categorization ──────────────────────────────
echo ""
echo "Confirming all uploads and triggering profile categorization ..."
CONFIRM_RESP=$(curl -sf -X POST "$API_URL/ingestion/confirm-all" \
  -H "$AUTH_HEADER" 2>&1) \
  || fail "confirm-all failed. Check API logs: docker compose logs api"
ok "Uploads confirmed. Categorization queued."
echo "  Response: $CONFIRM_RESP"

# ── Step 6: Wait for categorization ──────────────────────────────────────────
echo ""
echo "Waiting for profile categorization (up to 90s) ..."
for i in $(seq 1 45); do
  PROF_RESP=$(curl -sf "$API_URL/profile/status" \
    -H "$AUTH_HEADER" 2>&1) || { warn "Profile status poll failed"; break; }
  PROF_STATUS=$(echo "$PROF_RESP" | python3 -c "import sys,json; print(json.load(sys.stdin).get('status','unknown'))" 2>/dev/null || echo "unknown")
  if [ "$PROF_STATUS" = "done" ]; then
    ok "Profile categorization complete"
    break
  fi
  if [ "$i" -eq 45 ]; then
    warn "Profile categorization timed out - demo may still work if partial profile is available"
  else
    sleep 2
  fi
done

# ── Done ──────────────────────────────────────────────────────────────────────
echo ""
echo "╔════════════════════════════════════════════════════════╗"
echo "║  Demo seed complete!                                   ║"
echo "╠════════════════════════════════════════════════════════╣"
echo "║  Email:    $DEMO_EMAIL                    ║"
echo "║  Password: $DEMO_PASSWORD              ║"
echo "║  URL:      http://localhost:3000                       ║"
echo "╚════════════════════════════════════════════════════════╝"
echo ""
```

2. Make the script executable: `chmod +x scripts/seed-demo.sh`

3. **Check the auth response shape** - the script assumes the login/signup response nests tokens under `tokens.access_token`. Verify against the actual `AuthResponseDTO` in `api/app/schemas/auth.py`:
   - If the actual key is `access_token` (flat, not nested), adjust the `python3 -c` extraction line accordingly
   - Run `curl -sf -X POST http://localhost:8000/auth/signup -H "Content-Type: application/json" -d '{"email":"check@test.com","password":"Test1234!"}' | python3 -m json.tool` and update the script to match

**Verification:**
- `bash scripts/seed-demo.sh` runs to completion from a fresh DB state
- Output shows "Demo seed complete!" with credentials
- Log in at `http://localhost:3000` with `demo@senso.app` / `SensoDEMO2026!` → profile is visible

### Task 2: Create `scripts/reset-demo.sh`

**Files to change:**
- `scripts/reset-demo.sh` - new file, executable bash script

**Steps:**

1. **Create `scripts/reset-demo.sh`**:

```bash
#!/usr/bin/env bash
# reset-demo.sh - Wipe demo user data between demo runs
# Usage: bash scripts/reset-demo.sh
#
# Strategy: connect directly to the Postgres container and run SQL to delete
# the demo user's rows from all relevant tables. This is faster and more
# reliable than calling the API delete endpoints (which may require admin tokens).
#
# Tables cleared (in dependency order to avoid FK violations):
#   coaching_messages → coaching_sessions → ingestion_uploads → user_profiles → users
#
# The demo user account itself is also deleted so seed-demo.sh can re-create it.

set -euo pipefail

DEMO_EMAIL="${DEMO_EMAIL:-demo@senso.app}"
POSTGRES_USER="${POSTGRES_USER:-senso}"
POSTGRES_PASSWORD="${POSTGRES_PASSWORD:-senso}"
POSTGRES_DB="${POSTGRES_DB:-senso}"

GREEN='\033[0;32m'; YELLOW='\033[1;33m'; RED='\033[0;31m'; NC='\033[0m'
ok()   { echo -e "${GREEN}✓${NC} $*"; }
warn() { echo -e "${YELLOW}⚠${NC}  $*"; }
fail() { echo -e "${RED}✗${NC} $*"; exit 1; }

echo ""
echo "╔════════════════════════════════════════╗"
echo "║   S.E.N.S.O. Demo Reset Script         ║"
echo "╚════════════════════════════════════════╝"
echo ""
echo "Target demo email: $DEMO_EMAIL"
echo ""

# ── Confirm intent ────────────────────────────────────────────────────────────
read -rp "This will delete ALL data for $DEMO_EMAIL. Continue? [y/N] " CONFIRM
if [[ ! "$CONFIRM" =~ ^[Yy]$ ]]; then
  echo "Aborted."
  exit 0
fi

# ── Check docker compose is running ──────────────────────────────────────────
if ! docker compose ps postgres | grep -q "running\|Up"; then
  fail "Postgres container is not running. Run: docker compose up -d postgres"
fi

# ── Run SQL delete via psql in the postgres container ────────────────────────
echo ""
echo "Deleting demo user data from database ..."

docker compose exec -T postgres psql \
  -U "$POSTGRES_USER" \
  -d "$POSTGRES_DB" \
  -v ON_ERROR_STOP=1 \
  <<'SQL'
DO $$
DECLARE
  demo_email TEXT := current_setting('reset.demo_email', true);
  uid UUID;
BEGIN
  -- Find the user
  SELECT id INTO uid FROM users WHERE email = demo_email;
  IF uid IS NULL THEN
    RAISE NOTICE 'Demo user % not found - nothing to delete.', demo_email;
    RETURN;
  END IF;

  -- Delete in dependency order (child tables first)
  DELETE FROM coaching_messages
    WHERE session_id IN (
      SELECT id FROM coaching_sessions WHERE user_id = uid
    );
  RAISE NOTICE 'coaching_messages deleted';

  DELETE FROM coaching_sessions WHERE user_id = uid;
  RAISE NOTICE 'coaching_sessions deleted';

  DELETE FROM welcome_cache WHERE TRUE;  -- welcome cache is not user-keyed, wipe all
  RAISE NOTICE 'welcome_cache cleared';

  DELETE FROM ingestion_uploads WHERE user_id = uid;
  RAISE NOTICE 'ingestion_uploads deleted';

  DELETE FROM user_profiles WHERE user_id = uid;
  RAISE NOTICE 'user_profiles deleted';

  DELETE FROM users WHERE id = uid;
  RAISE NOTICE 'user deleted (id=%)', uid;

  RAISE NOTICE 'Reset complete for %', demo_email;
END;
$$ LANGUAGE plpgsql;
SQL

# psql doesn't easily pass shell variables into the DO block with current_setting.
# Instead pass the email as a SET command before the DO block:

# Re-run with email passed in:
docker compose exec -T postgres psql \
  -U "$POSTGRES_USER" \
  -d "$POSTGRES_DB" \
  -v ON_ERROR_STOP=1 \
  -c "SET reset.demo_email = '$DEMO_EMAIL';" \
  -c "
DO \$\$
DECLARE
  demo_email TEXT := current_setting('reset.demo_email', true);
  uid UUID;
BEGIN
  SELECT id INTO uid FROM users WHERE email = demo_email;
  IF uid IS NULL THEN
    RAISE NOTICE 'Demo user % not found.', demo_email;
    RETURN;
  END IF;
  DELETE FROM coaching_messages WHERE session_id IN (SELECT id FROM coaching_sessions WHERE user_id = uid);
  DELETE FROM coaching_sessions WHERE user_id = uid;
  DELETE FROM welcome_cache WHERE TRUE;
  DELETE FROM ingestion_uploads WHERE user_id = uid;
  DELETE FROM user_profiles WHERE user_id = uid;
  DELETE FROM users WHERE id = uid;
  RAISE NOTICE 'Reset complete for %', demo_email;
END;
\$\$ LANGUAGE plpgsql;
"

ok "Database reset complete for $DEMO_EMAIL"

# ── Also clear MinIO uploads for this user (best-effort) ──────────────────────
echo ""
echo "Clearing MinIO uploads (best-effort) ..."
docker compose exec -T minio mc alias set local http://localhost:9000 \
  "${MINIO_ROOT_USER:-minioadmin}" "${MINIO_ROOT_PASSWORD:-minioadmin}" 2>/dev/null || true

docker compose exec -T minio mc rm --recursive --force \
  "local/senso-uploads/" 2>/dev/null \
  && ok "MinIO bucket cleared" \
  || warn "MinIO clear failed (non-fatal - bucket may already be empty)"

# ── Done ──────────────────────────────────────────────────────────────────────
echo ""
echo "╔═════════════════════════════════════════════════════════╗"
echo "║  Reset complete. Run seed-demo.sh to re-seed the demo. ║"
echo "╚═════════════════════════════════════════════════════════╝"
echo ""
```

2. Make the script executable: `chmod +x scripts/reset-demo.sh`

3. **Verify actual table names** - check `api/app/db/models.py` for the real SQLAlchemy table names (`__tablename__`). The SQL in `reset-demo.sh` assumes:
   - `users`, `user_profiles`, `ingestion_uploads`, `coaching_sessions`, `coaching_messages`, `welcome_cache`
   Update any table names that differ from the actual model definitions.

4. **Verify `AuthResponseDTO` token shape** used in `seed-demo.sh` Task 1 step 3:
   - Read `api/app/schemas/auth.py` and check the shape of `AuthResponseDTO`
   - If tokens are nested (e.g., `response.tokens.access_token`), the seed script python extraction already handles this
   - If flat (e.g., `response.access_token`), update the extraction line in Task 1

**Verification:**
- `bash scripts/seed-demo.sh` → "Demo seed complete!" with credentials
- `bash scripts/reset-demo.sh` → "Reset complete" 
- `bash scripts/seed-demo.sh` again → second run succeeds (re-creates account cleanly)
- Full round-trip: seed → login at `http://localhost:3000` → profile visible → reset → seed again → profile visible

1. `bash scripts/seed-demo.sh` creates a demo user with email `demo@senso.app` and uploads at least 2 sample files, waits for extraction, confirms all, and waits for profile categorization.

3. `bash scripts/reset-demo.sh` (with `y` confirmation) deletes all user data and the account, and a subsequent `seed-demo.sh` run succeeds.

5. Scripts exist at `scripts/seed-demo.sh` and `scripts/reset-demo.sh` and are executable (`chmod +x`).
