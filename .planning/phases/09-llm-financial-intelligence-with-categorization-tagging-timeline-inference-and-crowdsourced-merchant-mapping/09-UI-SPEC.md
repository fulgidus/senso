## Spacing Scale

Declared values (must be multiples of 4). **Matches existing project convention** (`ProfileScreen.tsx`, `AppShell.tsx`).

| Token | Value | Usage                                  |
| ----- | ----- | -------------------------------------- |
| xs    | 4px   | Icon gaps, badge padding, pill gaps    |
| sm    | 8px   | Compact list row spacing, inline items |
| md    | 16px  | Default card padding, form gap         |
| lg    | 24px  | Section card padding (`p-6`)           |
| xl    | 32px  | Between major sections (`space-y-8`)   |
| 2xl   | 48px  | Page top padding                       |
| 3xl   | 64px  | Reserved for page-level hero areas     |

Exceptions:
- Timeline event connector dots: 12px diameter (3px border, centred on 24px row)
- Notification badge: 20px diameter (fixed, overlays bell icon)
- Touch targets for dismiss/blacklist icon buttons: minimum 44px (accessibility requirement)

> Source: Default 8-point scale confirmed against `ProfileScreen.tsx` patterns (p-5, p-6, gap-4, space-y-8).

## Component Inventory

New components introduced in Phase 9, with their visual contracts:

### 1. ProfileScreen — "Timeline" Tab

**Container:** New third tab alongside existing chart sections. Tab bar uses `rounded-full px-4 py-1.5 text-sm font-medium` pill style with `bg-primary text-primary-foreground` for active state.

**Timeline event card:**
```
rounded-2xl border border-border bg-card p-5
├── Event type icon (lucide, h-4 w-4, text-primary)
├── Date chip: rounded-full bg-secondary px-2 py-0.5 text-xs text-muted-foreground
├── Title: text-base font-semibold text-foreground
├── Description: text-sm text-muted-foreground
├── Evidence summary: text-xs text-muted-foreground (collapsed by default, expand on click)
├── "Add context" button: variant="ghost" text-sm text-primary (shows textarea inline)
└── "Dismiss" button: variant="ghost" text-sm text-muted-foreground (shows reason picker inline)
```

**Dismiss reason picker:** Inline `<select>` or radio group with values: `false_assumption` | `clerical_error` | `outdated` | `duplicate` | `other`. When "other" is selected, a short textarea (max 200 chars visible, enforced backend) appears. Confirm with a small "Conferma" button (`variant="default" size="sm"`).

**Add context textarea:** Full-width textarea, no visible character limit (backend enforces max), placeholder `"Aggiungi contesto a questo evento..."`. Save button `variant="default" size="sm"` labelled "Salva contesto".

**User context display (after save):** Distilled version shown as italic `text-sm text-muted-foreground` below description. Raw version never displayed.

**Dismissed events:** Shown collapsed at bottom of timeline list, behind a `"Mostra {{n}} eventi nascosti"` ghost button. Dismissed events render at 50% opacity with a strikethrough on the title.

**Empty state (no events detected):** Centred icon (`Clock` lucide, h-10 w-10 text-muted-foreground`) + heading "Nessun evento rilevato" + body "Man mano che carichi documenti, rileveremo eventi come cambi di lavoro, grandi acquisti e cambi di abbonamenti." No CTA.

### 2. Review Uncategorized Screen (`/profile/uncategorized`)

**Route:** New route, accessible from:
- ProfileScreen: amber warning banner CTA "Rivedi transazioni" when uncategorized > 0
- App nav: not a primary nav item (accessed only via CTA or notification)

**Layout:** Full-page, same `mx-auto w-full max-w-4xl px-6 py-6` container as ProfileScreen.

**Header area:**
```
h2 text-xl font-semibold: "Transazioni non categorizzate"
p text-sm text-muted-foreground: "{{count}} transazioni richiedono la tua attenzione"
```

**Transaction row:**
```
rounded-xl border border-border bg-card px-4 py-3 (NOT rounded-2xl — compact list)
├── Left: description text-sm font-medium text-foreground (truncate at 1 line)
│         date text-xs text-muted-foreground
│         source filename badge: rounded-full bg-secondary px-2 py-0.5 text-xs
├── Centre: amount text-sm font-semibold text-foreground (right-aligned)
└── Right: Category picker dropdown (shadcn Select or native <select>)
           Options: full VALID_CATEGORIES list, sorted alphabetically
           Placeholder: "Scegli categoria"
           On select: optimistic immediate save + visual confirmation (green checkmark icon fades out after 2s)
```

**Ordering:** Frequency-first (same description appearing N times grouped), then by absolute monthly impact. Group header for repeated descriptions: `text-xs text-muted-foreground` showing `"{{n}} occorrenze · €{{total}}/mese"`.

**Empty state:** `CheckCircle` lucide icon (h-10 w-10 text-primary) + "Tutte le transazioni sono categorizzate" + body text.

**Error state:** `AlertTriangle` icon + "Impossibile caricare le transazioni. Controlla la connessione e riprova." + "Riprova" ghost button.

## Interaction Contracts

### Category Picker (Uncategorized Review)
- **Input:** shadcn `Select` or native `<select>` for accessibility
- **On select:** Immediate optimistic update → `PATCH /profile/transactions/{id}/category`
- **Success:** Green `CheckCircle` icon replaces picker briefly (1.5s), then row becomes muted
- **Failure:** Revert to "Scegli categoria" placeholder + `text-destructive` error message inline

### Dismiss Event (Timeline)
- **Trigger:** Ghost "Nascondi evento" button
- **Expansion:** Inline, no modal — reason picker appears below description within same card
- **Confirm:** `POST /profile/timeline/{id}/dismiss` with `{ reason, detail? }`
- **Result:** Card animates to 50% opacity, title gets `line-through`, moves to "dismissed" section

### Add Context (Timeline)
- **Trigger:** Ghost "Aggiungi contesto" button
- **Expansion:** Inline textarea below description, autofocus
- **Character limit:** None shown visually (backend enforces; show error only if backend rejects)
- **Save:** `POST /profile/timeline/{id}/context` — shows spinner in button during save
- **After save:** Textarea replaced by distilled context (italic muted text) + "Modifica" ghost button

### Blacklist Merchant (Admin)
- **Trigger:** "Blacklista" button in admin table row
- **Expansion:** Inline row expansion (not modal) — textarea for reason (required, min 5 chars)
- **Confirm:** `POST /admin/learned-merchants/{id}/blacklist` — row immediately dims to 50% opacity
- **Unblacklist:** "Ripristina" ghost button on blacklisted rows

### Notification Bell
- **Mount:** Fetch `GET /notifications?limit=20` on auth shell mount
- **Poll:** Re-fetch every 30s while app is active (or on WebSocket message if future enhancement)
- **Mark read:** `POST /notifications/{id}/read` on panel item click
- **Mark all read:** `POST /notifications/read-all` on panel header button click
- **Badge:** Disappears when unread count reaches 0 (no animation required)
