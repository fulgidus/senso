## Rendering Hierarchy (Final)

Inside the chat message stream, the order for an assistant turn is:

```
1. [SEPARATE MESSAGES] Tool-usage pills (during streaming)
   ┌─ 🔍 Cercando nelle tue transazioni… ─┐  ← separate chat message
   └──────────────────────────────────────────┘
   ┌─ 📊 Caricando il tuo profilo… ──────────┐  ← separate chat message
   └──────────────────────────────────────────┘

2. [ASSISTANT BUBBLE] Main response (single bubble)
   ┌──────────────────────────────────────────┐
   │  [if purchase intent] Verdict pill        │  ← affordability_verdict
   │                                          │
   │  Coach message text                      │  ← message
   │                                          │
   │  [if transactions cited]                 │
   │  Transazioni citate                      │  ← transaction_evidence
   │  ┌────────┬──────────┬────────┐          │
   │  │ Data   │ Desc     │ Importo│          │
   │  └────────┴──────────┴────────┘          │
   │                                          │
   │  [if goal relevant]                      │
   │  🎯 Goal name         ~40%              │  ← goal_progress
   │  ████████████░░░░░░░░                    │
   │                                          │
   │  [if content found] ← horizontal scroll  │  ← content_cards
   │  ┌───────┐ ┌───────┐                    │
   │  │Card 1 │ │Card 2 │                    │
   │  └───────┘ └───────┘                    │
   │                                          │
   │  [if reminder emitted]                   │
   │  🔔 Reminder title          [CTA]       │  ← interactive_cards
   │                                          │
   │  [if STS mode]                           │
   │  Mostra dettagli ▾                       │  ← details_a2ui (collapsed)
   └──────────────────────────────────────────┘
```

**Rendering rules:**
- Each surface wraps in `{condition && <Component />}` — never render empty containers
- Tool-usage pills are separate `DisplayMessage` entries with `role: "tool"` or a new role marker
- Tool pills collapse to a single summary line after response completes (if `"granular"` mode)
- No surface renders unconditionally — all are gated on data presence + context conditions

## Copywriting Contract

| Element                           | Italian                                              | English                                           | i18n Key                        |
| --------------------------------- | ---------------------------------------------------- | ------------------------------------------------- | ------------------------------- |
| Show details toggle               | Mostra dettagli                                      | Show details                                      | `coaching.showDetails`          |
| Hide details toggle               | Nascondi                                             | Hide                                              | `coaching.hideDetails`          |
| Default reminder CTA              | Imposta promemoria                                   | Set reminder                                      | `coaching.reminder.defaultCta`  |
| Goal progress subtitle            | Stimato dal tuo margine mensile attuale              | Estimated from your current monthly margin        | `coaching.goalProgress.estimated` |

**Empty states:** No new empty states — all new surfaces are conditional enrichments within existing chat. If no enrichments fire, the bubble shows only the message text (which is the correct behavior per D-09).

**Error states:** If tool-usage SSE event fails to parse, silently skip the pill (no user-visible error for intermediate tool activity).

**Destructive actions:** None in this phase (no delete/remove operations added).
