# Plan 21-03 — details_a2ui Quality Gate

## Objective

Add `_validate_a2ui(panel) -> bool` to `CoachingService`. Strip `details_a2ui` from
the response if the panel has fewer than 2 data rows, all-null/zero values, or generic
labels. Update the schema to enforce `minimum: 2` rows. Prevents the current "1 number
rendered as a comparison panel" noise.

---

<task id="21-03-01" title="Implement _validate_a2ui quality gate">
  <objective>
    Add validation logic that nulls out `details_a2ui` panels that don't meet the
    minimum quality bar for a useful comparison panel.
  </objective>
  <read_first>
    - api/app/coaching/service.py — response assembly, existing schema repair logic
    - api/app/coaching/schemas/coaching_response.schema.json — details_a2ui structure
  </read_first>
  <action>
    Add to `CoachingService`:
    ```python
    def _validate_a2ui(self, panel: dict | None) -> bool:
        """Return True only if panel is worth rendering."""
        if not panel:
            return False
        rows = panel.get("rows", [])
        if len(rows) < 2:
            return False
        # All values null or zero
        values = [r.get("value") for r in rows]
        if all(v in (None, 0, "0", "", "—") for v in values):
            return False
        # Generic labels (LLM placeholder pattern)
        labels = [str(r.get("label", "")).lower() for r in rows]
        generic = {"valore1", "valore2", "item1", "item2", "label", "value"}
        if all(l in generic for l in labels):
            return False
        return True
    ```

    In response assembly:
    ```python
    if not self._validate_a2ui(response.get("details_a2ui")):
        response["details_a2ui"] = None
    ```
  </action>
  <acceptance_criteria>
    - Panel with 1 row → nulled out
    - Panel with 2 rows, one null value → nulled out (all-null/zero check)
    - Panel with 2 rows, real values → kept
    - Panel with generic labels "valore1"/"valore2" → nulled out
    - Pytest: `test_a2ui_gate.py` with 6 cases covering all branches
  </acceptance_criteria>
</task>
