# Plan 21-04 - Chat UI Rendering Hierarchy

## Objective

Fix the chat UI rendering order and visual structure: verdict → key message → resource
cards (horizontal strip) → action cards → details panel (collapsed). Prevent empty card
containers from rendering. Add horizontal scroll strip for cards on mobile. Add
`"Mostra dettagli ▾"` toggle for `details_a2ui`.

---

<task id="21-04-01" title="Enforce rendering hierarchy + no empty containers">
  <objective>
    Update `AssistantBubble` (or equivalent response renderer in ChatScreen) to follow
    the surface rendering order and never render empty wrappers.
  </objective>
  <read_first>
    - senso/src/features/coaching/ChatScreen.tsx - AssistantBubble or inline response
      rendering, current card render order
    - senso/src/features/coaching/AssistantBubble.tsx - if it exists separately
  </read_first>
  <action>
    Rendering order (top to bottom in bubble):
    1. `affordability_verdict` badge (only if not null) - green/yellow/red pill
    2. `message` text
    3. Resource cards (only if `resource_cards.length > 0`) - horizontal scroll strip
    4. Action cards (only if `action_cards.length > 0`) - vertical stack or horizontal
    5. Details panel (only if `details_a2ui` not null) - collapsed by default

    For each surface: wrap in `{condition && <Component />}` - never `{condition ?
    <Component /> : <div />}` or rendering empty containers.

    Resource cards horizontal strip:
    ```tsx
    <div className="flex gap-2 overflow-x-auto pb-1 -mx-1 px-1 snap-x">
      {resource_cards.map(c => <ResourceCard key={c.id} card={c} className="snap-start shrink-0 w-56" />)}
    </div>
    ```

    Action cards: `className="flex flex-col gap-2"` (vertical stack, distinct visual).
  </action>
  <acceptance_criteria>
    - Verdict badge only appears when affordability_verdict is not null
    - Resource cards scroll horizontally on mobile (overflow-x-auto)
    - Empty card arrays produce no DOM elements (inspect reveals no empty divs)
    - Rendering order matches: verdict → message → resources → actions → details
  </acceptance_criteria>
</task>

  <objective>
    Wrap `details_a2ui` in a collapsible with "Mostra dettagli ▾" / "Nascondi ▴"
    toggle. Give action cards a visually distinct style from resource cards (icon + CTA
    button, not thumbnail card).
  </objective>
  <read_first>

    - senso/src/locales/it.json - for i18n keys

  <action>
    Details panel toggle:
    ```tsx
    const [showDetails, setShowDetails] = useState(false);
    // ...
    {details_a2ui && (
      <div>
        <button onClick={() => setShowDetails(s => !s)}
          className="text-sm text-muted-foreground flex items-center gap-1">
          {showDetails ? t('coaching.hideDetails') : t('coaching.showDetails')}
          <span>{showDetails ? '▴' : '▾'}</span>
        </button>
        {showDetails && <A2UIPanel panel={details_a2ui} />}
      </div>
    )}
    ```

    Add i18n keys:
    - `coaching.showDetails`: "Mostra dettagli" (it) / "Show details" (en)

    Action card visual: use `bg-primary/10 border border-primary/20 rounded-lg p-3`
    with an icon (`Zap` from lucide) + a `<Button size="sm">` for the CTA. Distinct
    from resource card thumbnail style.
  </action>

    - Action cards have icon + CTA button (visually distinct from resource cards)
    - i18n keys present in both it.json and en.json
  </acceptance_criteria>
</task>
