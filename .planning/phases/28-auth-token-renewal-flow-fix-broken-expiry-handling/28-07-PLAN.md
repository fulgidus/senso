# Plan 28-07: Consumer wiring — auth layer, routing, settings, and direct call migrations

## Objective
Wire `onUnauthorized` into the 7 files that form the auth layer, routing shell, and
settings surface. This plan specifically:

1. **`useAuth.ts`** — replaces the raw `pollMessages()` call with `createMessagesApi(onUnauthorized).pollMessages()`
2. **`ChatRoutes.tsx`** — adds factory instantiation for `getProfile` and `listSessions`
3. **`RootResolver.tsx`** — adds factory for `getProfileStatus` and `getProfile`
4. **`AppShell.tsx`** — replaces `getNotifications` with factory call
5. **`OnboardingRoutes.tsx`** — migrates raw `apiRequest` confirm-all call to `createIngestionApi(onUnauthorized).confirmAll()`; **removes** the direct `apiRequest` import
6. **`SettingsScreen.tsx`** — migrates `getPersonas` and claim-handle calls to factories; **removes** dynamic `apiRequest` imports
7. **`AdminHandleGateModal.tsx`** — migrates claim-handle call to factory; **removes** dynamic `apiRequest` import

After this plan, `D-07` (component direct calls → factory migration) is satisfied for
these 7 files.

## must_haves
- [ ] `useAuth.ts` uses `createMessagesApi(onUnauthorized).pollMessages()` — no raw `pollMessages()` import
- [ ] `ChatRoutes.tsx` uses `createProfileApi` and `createCoachingApi` via `useMemo`
- [ ] `RootResolver.tsx` uses `createProfileApi` via `useMemo`
- [ ] `AppShell.tsx` uses `createNotificationsApi` via `useMemo`
- [ ] `OnboardingRoutes.tsx` uses `createIngestionApi(onUnauthorized).confirmAll(token)` — no direct `apiRequest` import remaining
- [ ] `SettingsScreen.tsx` uses `createCoachingApi` for `getPersonas` and `createAdminMerchantApi` for claim-handle — no dynamic `apiRequest` imports remaining
- [ ] `AdminHandleGateModal.tsx` uses `createAdminMerchantApi` for claim-handle — no dynamic `apiRequest` import
- [ ] `docker compose run --rm frontend pnpm tsc --noEmit` exits 0
