# Plan 28-01: AuthContext extension + session.ts exclusion comments

## Objective
Add `onUnauthorized: () => Promise<string | null>` to `AuthContextValue` so every
`useAuthContext()` consumer can access the callback. Pass `auth.onUnauthorized` from
`App.tsx` into `AuthContext.Provider`. Add explicit `/* no onUnauthorized — auth primitive */`
comments to all `apiRequest` calls in `session.ts` that must not trigger refresh/redirect.

This plan is the prerequisite for all Wave 2 and Wave 3 plans. Nothing else can wire
`onUnauthorized` through context until this is done.

## must_haves
- [ ] `AuthContextValue` type in `AuthContext.tsx` includes `onUnauthorized: () => Promise<string | null>`
- [ ] `App.tsx` `<AuthContext.Provider>` passes `onUnauthorized: auth.onUnauthorized`
- [ ] `session.ts` has at least 2 `/* no onUnauthorized — auth primitive */` comments on `getMe` and `updateMe` calls
- [ ] `session.ts` `refresh()` already has a similar note; that note is preserved
- [ ] `docker compose run --rm frontend pnpm tsc --noEmit` exits 0


**Step 2 — `senso/src/App.tsx`:**

In the `<AuthContext.Provider value={{...}}>` block (the object passed to `value=`),
add `onUnauthorized: auth.onUnauthorized,` immediately after
`updateUser: auth.updateUser,`. The block currently looks like:


After the edit it should contain:
```typescript
                signOut: auth.signOut,
                updateUser: auth.updateUser,
```

### Task 28-01-02: Add exclusion comments to session.ts auth-primitive calls

<read_first>
- senso/src/features/auth/session.ts
</read_first>

<action>
In `senso/src/features/auth/session.ts`, add `/* no onUnauthorized — auth primitive */`
comments to each `apiRequest` call inside auth-primitive functions. Do NOT add a factory
pattern to this file — these calls intentionally lack `onUnauthorized`.

**1. Inside `refresh()` function** — already has a note like
`// NOTE: No onUnauthorized here - prevents infinite refresh loops.`
Upgrade it to use the canonical comment format so the grep check passes:

Find the line inside `refresh()` that reads:
```typescript
    // NOTE: No onUnauthorized here - prevents infinite refresh loops.
```
Replace it with:
```typescript
    /* no onUnauthorized — auth primitive, 401 here means refresh loop; see makeOnUnauthorized */
```

**2. Inside `getMe()` function** — find the `apiRequest<{ user: RawUser }>` call and add
the comment on the line immediately before it:
```typescript
    /* no onUnauthorized — auth primitive */
    const raw = await apiRequest<{ user: RawUser }>(backendBaseUrl, "/auth/me", {
```

**3. Inside `updateMe()` function** — find the `apiRequest<RawUser>` call and add the
comment on the line immediately before it:
```typescript
    /* no onUnauthorized — auth primitive */
    const raw = await apiRequest<RawUser>(backendBaseUrl, "/auth/me", {
```

**`signup()`, `login()`, `signout()`** — these functions call `apiRequest` without a
token (public endpoints) or swallow all errors (`signout`). They do not need the comment.
`startGoogle()` is also a public call — no comment needed.
</action>

<acceptance_criteria>
- `grep -c "no onUnauthorized — auth primitive" senso/src/features/auth/session.ts` outputs `3` (refresh + getMe + updateMe)
- `grep "onUnauthorized" senso/src/features/auth/session.ts | grep -v "no onUnauthorized"` returns 0 matches (confirms no actual parameter usage)
- `docker compose run --rm frontend pnpm tsc --noEmit` exits 0
</acceptance_criteria>
