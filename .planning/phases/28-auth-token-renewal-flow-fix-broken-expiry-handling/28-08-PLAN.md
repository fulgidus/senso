# Plan 28-08: Consumer wiring — feature screens and admin pages

## Objective
Wire `onUnauthorized` into all feature-level screens and admin pages. Each consumer
follows the `useMemo(() => createXxxApi(onUnauthorized), [onUnauthorized])` pattern
mandated by D-04.

This plan covers:
1. **`ChatScreen.tsx`** — coachingApi consumers (sendMessage, getWelcomeMessage, listSessions, etc.)
2. **`ProfileScreen.tsx`** — profile-api consumers (getProfile, getProfileStatus, etc.)
3. **`useProfileStatus.ts`** — profile-api hook (adds useAuthContext + factory)
4. **`QuestionnaireScreen.tsx`** — profile-api consumer (submitQuestionnaire)
5. **`TimelineTab.tsx`** — profile-api consumers (getTimeline, dismissTimelineEvent, addTimelineContext)
6. **`UncategorizedScreen.tsx`** — profile-api consumers (getUncategorized, bulkUpdateCategoryByDescription)
7. **`FilesTab.tsx`** — ingestionFilesApi consumers (listUploads, deleteUpload, retryUpload)
8. **`AdminInspectorDrawer.tsx`** — ingestionFilesApi consumers (getExtracted, getTrace)
9. **`ContentAdminPage.tsx`** — adminContentApi consumers
10. **`MerchantMapAdminPage.tsx`** — adminMerchantApi consumers (getMerchantMap, blacklist*)
11. **`ModerationQueuePage.tsx`** — adminMerchantApi consumers (getModerationQueue, etc.)

## must_haves
- [ ] All 11 files use factory instances via `useMemo(() => createXxxApi(onUnauthorized), [onUnauthorized])`
- [ ] All direct standalone function imports replaced with factory imports
- [ ] `onUnauthorized` sourced from `useAuthContext()` in every component/hook (not prop-drilled)
- [ ] `docker compose run --rm frontend pnpm tsc --noEmit` exits 0
