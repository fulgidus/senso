# Plan 33-01: Domain Manifest — Pydantic Models

## Goal

Create the `be/app/domain/` package and define all Pydantic v2 models that represent a valid Domain Manifest. This is the schema contract that everything in v2.0 depends on — every subsequent phase reads from these models.

## must_haves

- `be/app/domain/__init__.py` exists (package marker)
- `be/app/domain/manifest.py` defines: `ChestDef`, `StorageConfig`, `ExtractorDef`, `PersonasConfig`, `BehaviorConfig`, `FilterDef`, `FilterPipelineConfig`, `EnrichmentCaps`, `EnrichmentConfig`, `UIComponent`, `UIConfig`, `DomainMeta`, `DomainManifest`
- All models use `model_config = ConfigDict(frozen=True)` (manifests are read-only after startup)
- `ChestDef.access` is a `frozenset[Literal["searchable", "tool_callable", "direct"]]`
- `ChestDef.scope` is `Literal["user", "domain", "platform"]`
- `StorageConfig.encrypted` defaults to `False` (PII feature flag — opt-in, not opt-out)
- `DomainManifest` has all top-level sections: `domain`, `chests`, `extractors`, `personas`, `behavior`, `filters`, `enrichment`, `ui`
- All models importable via `from app.domain.manifest import DomainManifest`
- `docker compose run --rm api python -c "from app.domain.manifest import DomainManifest; print('ok')"` exits 0

Create `be/app/domain/manifest.py` with the following complete content:

```python
"""
Domain Manifest — Pydantic v2 models.

A Domain Manifest is a YAML file that declaratively configures the SENSO platform
for a specific advisory domain (finance, health, legal, etc.).  Loading a manifest
replaces all hardcoded domain logic with runtime configuration.

Design rules:
  - All models are frozen (immutable after startup load).
  - Optional fields default to safe/empty values so manifests can start minimal.
  - `ChestDef` is the core primitive; every other section references or extends it.
  - No I/O in this module — pure data models only.
"""

from __future__ import annotations

from typing import Any, Literal

from pydantic import BaseModel, ConfigDict, Field


# ──────────────────────────────────────────────────────────────────────────────
# Storage
# ──────────────────────────────────────────────────────────────────────────────


class StorageConfig(BaseModel):
    """Per-chest storage behaviour."""

    model_config = ConfigDict(frozen=True)

    encrypted: bool = False
    """PII feature flag — when True the chest uses EncryptedJSON columns."""

    localized: bool = False
    """When True, chest content is locale-scoped (language-specific rows)."""

    ttl_days: int | None = None
    """Auto-expire entries after N days.  None = permanent."""


# ──────────────────────────────────────────────────────────────────────────────
# Chest
# ──────────────────────────────────────────────────────────────────────────────

AccessMode = Literal["searchable", "tool_callable", "direct"]
ChestScope = Literal["user", "domain", "platform"]


class ChestDef(BaseModel):
    """
    A typed, self-contained knowledge container.

    Chests are the core abstraction in v2.0.  Every piece of domain data (user
    profiles, transactions, content, coaching insights) is a Chest with explicit
    schema, scope, access rules, and storage config.
    """

    model_config = ConfigDict(frozen=True)

    id: str
    """Unique identifier within the domain (e.g. 'user_profile', 'transactions')."""

    type: str
    """Semantic type label used by platform subsystems to select typed accessors."""

    scope: ChestScope = "user"
    """
    user     — private to a single authenticated user.
    domain   — shared across all users of this domain (content catalog, regional data).
    platform — global across all domains (reserved for v2.1+).
    """

    access: frozenset[AccessMode] = frozenset({"tool_callable"})
    """
    searchable    — indexed in BM25 at startup; available via search tool calls.
    tool_callable — LLM can call a generated tool to read/write this chest.
    direct        — accessed programmatically only; not exposed to LLM tools.
    """

    storage: StorageConfig = Field(default_factory=StorageConfig)
    """Storage behaviour including encryption, localisation, and TTL."""

    schema_ref: str | None = None
    """
    Path to a JSON Schema file (relative to the manifest directory).
    Used by DataLayer for JSONB validation and by tool generator for parameter docs.
    """

    tool_spec: dict[str, Any] | None = None
    """
    Optional override for auto-generated LLM tool definition.
    If None, tool is generated from schema_ref + chest metadata.
    Populated by Phase 35 (AI Tool Generation).
    """


# ──────────────────────────────────────────────────────────────────────────────
# Extractors
# ──────────────────────────────────────────────────────────────────────────────

ExtractorType = Literal["module", "adaptive", "webhook"]


class ExtractorDef(BaseModel):
    """An ingestion extractor that routes a document type into one or more Chests."""

    model_config = ConfigDict(frozen=True)

    id: str
    """Unique extractor ID."""

    type: ExtractorType = "module"
    """
    module   — a builtin Python module with FINGERPRINT + extract() interface.
    adaptive — LLM-driven adaptive pipeline (no builtin module required).
    webhook  — inbound webhook payload routed to Chests.
    """

    module_ref: str | None = None
    """Python dotted-path to the module (only for type='module')."""

    document_types: list[str] = Field(default_factory=list)
    """Human-readable document type tags (e.g. ['bank_csv', 'payslip'])."""

    target_chests: list[str] = Field(default_factory=list)
    """IDs of Chests that this extractor writes into."""


# ──────────────────────────────────────────────────────────────────────────────
# Personas
# ──────────────────────────────────────────────────────────────────────────────


class PersonasConfig(BaseModel):
    """Pointer to the domain's persona configuration file."""

    model_config = ConfigDict(frozen=True)

    config_ref: str
    """
    Path to personas config file (JSON or YAML) relative to manifest directory.
    The file must define a list of personas with id, name, soul_ref, etc.
    Format follows the existing be/app/personas/config.json structure.
    """


# ──────────────────────────────────────────────────────────────────────────────
# Behavior
# ──────────────────────────────────────────────────────────────────────────────


class BehaviorConfig(BaseModel):
    """Paths to domain-specific behavior directive files."""

    model_config = ConfigDict(frozen=True)

    ethos_ref: str | None = None
    """Ethos markdown: coaching philosophy and tone."""

    boundaries_ref: str | None = None
    """Boundaries markdown: soft off-limit topics (LLM-enforced via prompt)."""

    allowlist_ref: str | None = None
    """Allowlist markdown: topics explicitly permitted even if borderline."""

    hard_boundaries_ref: str | None = None
    """
    hard-boundaries.yml: regex-based safety rules for input/output scanning.
    Loaded by SafetyScanner at startup.
    """


# ──────────────────────────────────────────────────────────────────────────────
# Filters
# ──────────────────────────────────────────────────────────────────────────────


class FilterDef(BaseModel):
    """A single filter stage in an input or output pipeline."""

    model_config = ConfigDict(frozen=True)

    type: str
    """
    Built-in filter types:
      safety_scan      — SafetyScanner regex rules from hard_boundaries_ref
      pii_scrub        — strip PII tokens from LLM-visible content
      domain_compliance — enforce domain-specific allow/block rules
    Custom types are resolved via pluggy hooks (Phase 36).
    """

    config: dict[str, Any] = Field(default_factory=dict)
    """Filter-specific configuration parameters."""


class FilterPipelineConfig(BaseModel):
    """Ordered input and output filter pipelines."""

    model_config = ConfigDict(frozen=True)

    input: list[FilterDef] = Field(default_factory=list)
    """Filters applied to user messages before LLM processing."""

    output: list[FilterDef] = Field(default_factory=list)
    """Filters applied to LLM responses before delivery to client."""


# ──────────────────────────────────────────────────────────────────────────────
# Enrichment
# ──────────────────────────────────────────────────────────────────────────────


class EnrichmentCaps(BaseModel):
    """Hard caps on enrichment fields in coaching responses."""

    model_config = ConfigDict(frozen=True)

    content_cards: int = 3
    interactive_cards: int = 2
    evidence_rows: int = 5
    goal_progress: int = 2


class EnrichmentConfig(BaseModel):
    """Domain-configurable response enrichment schema and caps."""

    model_config = ConfigDict(frozen=True)

    schema_ref: str | None = None
    """Path to JSON Schema file defining the enriched coaching response format."""

    caps: EnrichmentCaps = Field(default_factory=EnrichmentCaps)
    """Hard limits on how many cards / rows the LLM can include."""


# ──────────────────────────────────────────────────────────────────────────────
# UI
# ──────────────────────────────────────────────────────────────────────────────


class UIComponent(BaseModel):
    """A SDUI component descriptor declared by the domain."""

    model_config = ConfigDict(frozen=True)

    type: str
    """Component type key (e.g. 'affordability_card', 'transaction_evidence')."""

    renderer: Literal["sdui", "custom"] = "sdui"
    """
    sdui   — rendered by the SDUI engine from domain catalog specs.
    custom — rendered by domain-specific React component (Phase 38 fallback).
    """

    config: dict[str, Any] = Field(default_factory=dict)
    """Renderer-specific configuration."""


class UIConfig(BaseModel):
    """Domain-driven UI descriptor block."""

    model_config = ConfigDict(frozen=True)

    sdui_catalog_ref: str | None = None
    """Path to SDUI catalog YAML/JSON (Phase 38). None until SDUI is wired."""

    components: list[UIComponent] = Field(default_factory=list)
    """Declared component types. Planner uses this to know which UI elements exist."""


# ──────────────────────────────────────────────────────────────────────────────
# Domain metadata
# ──────────────────────────────────────────────────────────────────────────────


class DomainMeta(BaseModel):
    """Top-level domain identity block."""

    model_config = ConfigDict(frozen=True)

    id: str
    """Machine identifier (e.g. 'finance', 'health'). Must be slug-safe."""

    name: str
    """Human-readable domain name."""

    version: str = "1.0.0"
    """Semantic version of this domain manifest."""

    locale_default: str = "it"
    """Default locale for this domain."""


# ──────────────────────────────────────────────────────────────────────────────
# Root manifest
# ──────────────────────────────────────────────────────────────────────────────


class DomainManifest(BaseModel):
    """
    The complete Domain Manifest — root model loaded from YAML at startup.

    Every platform subsystem reads configuration from here rather than
    hardcoded files.  Frozen after load.
    """

    model_config = ConfigDict(frozen=True)

    domain: DomainMeta
    chests: list[ChestDef] = Field(default_factory=list)
    extractors: list[ExtractorDef] = Field(default_factory=list)
    personas: PersonasConfig | None = None
    behavior: BehaviorConfig = Field(default_factory=BehaviorConfig)
    filters: FilterPipelineConfig = Field(default_factory=FilterPipelineConfig)
    enrichment: EnrichmentConfig = Field(default_factory=EnrichmentConfig)
    ui: UIConfig = Field(default_factory=UIConfig)
```
