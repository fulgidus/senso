# Plan 33-03: FastAPI Startup Integration + Finance Stub Manifest

## Goal

Wire the Domain Manifest loading and Chest Registry initialization into FastAPI's lifespan so the platform configures itself from the manifest at startup. Create the Finance domain stub manifest that declares all 6 chest types — this must load successfully, satisfying Phase 33 success criterion 5.

## must_haves

- `be/app/main.py` lifespan calls `load_manifest(settings.domain_manifest_path)` and builds `ChestRegistry`; errors are fatal (startup refuses to proceed)
- `app.state.manifest` and `app.state.chest_registry` are available after startup
- A `ManifestLoadError` or `ChestRegistry.validate()` failure causes the app to log error + raise `RuntimeError` (not silently continue)
- `be/domains/finance/manifest.yaml` exists and is valid YAML that passes `DomainManifest.model_validate()`
- Finance manifest declares exactly 6 chests: `user_profile`, `transactions`, `content`, `regional_knowledge`, `coaching_insights`, `timeline_events`
- `docker compose run --rm api python -c "from app.domain.loader import load_manifest; m=load_manifest('domains/finance/manifest.yaml'); print(m.domain.id)"` prints `finance` and exits 0
- `docker compose up api` starts without errors related to manifest loading (check logs for `Domain manifest loaded:`)

<action>
In `be/app/main.py`, inside the `lifespan` async context manager, add manifest loading after `create_tables()` and before the scheduler setup.

Add these imports near the top of main.py (with other imports):
```python
from pathlib import Path
from app.domain.loader import load_manifest, ManifestLoadError
from app.domain.registry import ChestRegistry
```

Inside the `lifespan` function, after the `create_tables()` call, insert:
```python
    # ── Domain Manifest (v2.0) ─────────────────────────────────────────────
    _settings = get_settings()
    _manifest_path = Path(_settings.domain_manifest_path)
    try:
        _manifest = load_manifest(_manifest_path)
    except ManifestLoadError as exc:
        logger.error("Domain manifest load failed: %s", exc)
        raise RuntimeError(str(exc)) from exc

    _registry = ChestRegistry(_manifest, manifest_dir=_manifest_path.parent)
    _validation_errors = _registry.validate()
    if _validation_errors:
        for err in _validation_errors:
            logger.error("ChestRegistry: %s", err)
        raise RuntimeError(
            f"Domain manifest validation failed with {len(_validation_errors)} error(s). "
            f"First: {_validation_errors[0]}"
        )

    app.state.manifest = _manifest
    app.state.chest_registry = _registry
    logger.info(
        "Domain manifest loaded: %s v%s — %d chest(s)",
        _manifest.domain.name,
        _manifest.domain.version,
        len(_registry.ids()),
    )
```

The `yield` must remain after this block (existing code structure stays intact).
</action>
