## Layout & Interaction Contract

### Overlay

```
fixed inset-0 z-[9999]
bg-black/50 backdrop-blur-sm
flex items-center justify-center
overflow-y-auto p-4
```

- `z-[9999]` — above all existing `z-50` layers (AppShell drawer, dialogs)
- No `pointer-events-none` — overlay must trap pointer events
- `overflow-y-auto p-4` — allows scroll on very short screens

**Primary visual focal point:** the `ShieldCheck` icon + modal title in the header; the full-width Save button is the primary interaction anchor.

**Source:** `RecoveryPhraseInterstitial.tsx` uses `z-[9999] bg-background/95 backdrop-blur-sm`. This phase uses `bg-black/50` (same as `dialog.tsx` overlay) for visual consistency with the existing `shadcn Dialog` pattern.

### Modal Card

```
w-full max-w-sm
rounded-2xl border border-border
bg-card shadow-2xl
```

- **No close button** (`X`). Non-dismissable by design (D-04).
- **No `onEscapeKeyDown`** / **no `onPointerDownOutside`** dismissal. If using Radix primitives, both must be prevented with `e.preventDefault()`.
- `max-w-sm` (384px) — same as `dialog.tsx` content max-width. Sufficient for a single input field.

### Modal Structure (top to bottom)

```
┌─────────────────────────────────────────────┐
│  Header (border-b border-border px-6 py-5)  │
│  [ShieldCheck icon 20px, text-primary]       │
│  Title (text-base font-semibold)             │
│  Description (text-sm text-muted-foreground) │
├─────────────────────────────────────────────┤
│  Body (px-6 py-5 flex flex-col gap-4)       │
│                                              │
│  Input group:                                │
│  ┌──────────────────────────────────────┐   │
│  │ [!]  [ your-handle             ]     │   │
│  └──────────────────────────────────────┘   │
│  Format hint  (text-xs text-muted-fg)        │
│  Char count   (text-xs text-muted-fg right)  │
│                                              │
│  Error message (text-xs text-destructive)    │
│  — shown only when error state is non-null   │
├─────────────────────────────────────────────┤
│  Footer (px-6 pb-6)                         │
│  [Save handle button — full width]           │
└─────────────────────────────────────────────┘
```

### Input Prefix Group

The `!` prefix is rendered as a visually attached non-editable badge to the left of the text input:

```
border border-border rounded-md flex items-center overflow-hidden
  span.prefix: bg-muted px-3 py-2 text-sm font-semibold text-muted-foreground select-none
              content: "!"
  input:       flex-1 bg-transparent px-3 py-2 text-sm outline-none
              placeholder="mio-handle" (IT) / "my-handle" (EN)
              maxLength={30}
              pattern="[a-z0-9-]+"
              autoCapitalize="none"
              autoCorrect="off"
              spellCheck={false}
```

Focus state: the outer wrapper gets `ring-2 ring-ring ring-offset-0` when input is focused (apply via `focus-within:ring-2 focus-within:ring-ring`).
Error state: `border-destructive ring-destructive/20` on the outer wrapper.

**Why not use native `<input prefix>` pattern?** Consistent with how `SettingsScreen.tsx` renders the admin handle field — inline span + input flex group.

### Save Button

```
variant="default" size="lg" className="w-full"
disabled={saving || !inputValue.trim() || inputValue.trim().length < 3}
```

Loading state: replace button text with `<Loader2 className="h-4 w-4 animate-spin" />` + saving copy.

## Validation & Error States

| Condition                | Frontend validation | Error copy key                          |
| ------------------------ | ------------------- | --------------------------------------- |
| Input empty              | `disabled` button — no error shown | —                           |
| `< 3` chars              | `disabled` button — no error shown (format hint visible) | —          |
| Invalid chars (not `[a-z0-9-]`) | Filtered on `onChange` — invalid chars silently stripped | — |
| `> 30` chars             | `maxLength={30}` prevents entry         | —                                       |
| Reserved handle (`admin`, `sistema`, `senso`) | Backend 422 → error shown | `adminHandleGate.errorReserved` |
| Handle already taken     | Backend 409 → error shown              | `adminHandleGate.errorTaken`            |
| Network / other error    | Catch block → generic error            | `adminHandleGate.errorFallback`         |

**Inline vs tooltip:** Error displayed as inline `<p>` below the input group (`text-xs text-destructive mt-1`). Not a tooltip — consistent with how `SettingsScreen.tsx` renders `handleError`.

## Accessibility Contract

| Requirement             | Implementation                                                                      |
| ----------------------- | ----------------------------------------------------------------------------------- |
| `role="dialog"`         | Set on the outermost wrapper element                                                |
| `aria-modal="true"`     | Set on the outermost wrapper element — prevents screen reader from reading bg content |
| `aria-label`            | Use `t("adminHandleGate.dialogLabel")` on the dialog wrapper                       |
| `aria-labelledby`       | Title `<h2>` gets `id="admin-gate-title"`; dialog wrapper gets `aria-labelledby="admin-gate-title"` |
| `aria-describedby`      | Description `<p>` gets `id="admin-gate-desc"`; dialog wrapper gets `aria-describedby="admin-gate-desc"` |
| Focus trap              | On mount: `autoFocus` on the text input (not the button)                           |
| Focus on error          | After a failed save attempt: re-focus the text input                               |
| Screen reader live region | Error message rendered in `aria-live="polite"` container (always in DOM, empty when no error) |
| Input label             | `<label htmlFor="admin-handle-input">` with `className="sr-only"` using `adminHandleGate.inputLabel` |
| Escape key              | Does NOT close the modal (non-dismissable per D-04). Suppress via `onKeyDown` guard or Radix `onEscapeKeyDown={e => e.preventDefault()}` |
| No backdrop dismiss     | Suppress pointer events on overlay from closing modal (plain `<div>` overlay, not Radix `DialogOverlay`) |

## Gate Logic Contract (AppShell)

```tsx
// AppShell.tsx — before the main return(...)
const { user, updateUser } = useAuthContext();

if (user && user.isAdmin && !user.adminHandle) {
  return (
    <AdminHandleGateModal
      onSaved={(handle) => updateUser({ adminHandle: handle })}
    />
  );
}

// ... existing return (...) follows
```

- **Only rendered for:** `user.isAdmin === true && (!user.adminHandle || user.adminHandle === "")`
- **Dismissed by:** calling `onSaved(handle)` which triggers `updateUser({ adminHandle: handle })` — no page reload, no route change
- **Never rendered for:** non-admin users, or admins who already have a handle
