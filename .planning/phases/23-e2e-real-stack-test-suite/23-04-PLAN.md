# Plan 23-04 — Error Path E2E (Real Stack)

## Objective

Test error paths against the real running stack: expired token redirect, bad file
rejection, LLM timeout handling (not spinner forever), and network offline indicator.
These validate error handling that mocked tests can't catch.

---

<task id="23-04-01" title="Token expiry + bad file upload error paths">
  <objective>
    Test that expired auth token redirects to login, and that uploading an unsupported
    file type returns a proper error (not a 500).
  </objective>
  <read_first>
    - senso/e2e/real-stack/fixtures.ts — account fixture
    - api/app/routers/ingestion.py — file type validation, error codes
    - senso/src/features/auth/ — token storage, 401 handling
  </read_first>
  <action>
    Create `senso/e2e/real-stack/error-paths.spec.ts`:

    ```ts
    import { test, expect } from './fixtures';

    test('expired token redirects to login', async ({ page, account }) => {
      await loginAs(page, account);
      await page.goto('/chat'); // confirm logged in

      // Manually corrupt/expire the token in localStorage
      await page.evaluate(() => {
        const key = Object.keys(localStorage).find(k => k.includes('token') || k.includes('session'));
        if (key) localStorage.setItem(key, 'expired.invalid.token');
      });

      // Make an authenticated request — should trigger 401 → redirect to login
      await page.reload();
      // The 401 interceptor should redirect to /login
      await expect(page).toHaveURL(/login/, { timeout: 10_000 });
    });

    test('unsupported file type returns proper error', async ({ page, account }) => {
      await loginAs(page, account);
      await page.goto('/profilo');

      // Try to upload an .exe file
      const fileInput = page.locator('input[type="file"]');
      // Create a minimal .exe-like file in memory
      await fileInput.setInputFiles({
        name: 'malware.exe',
        mimeType: 'application/octet-stream',
        buffer: Buffer.from('MZ\x90\x00'), // minimal PE header
      });

      // Expect error message, not a 500 crash
      await expect(page.getByText(/formato non supportato|unsupported|tipo di file/i))
        .toBeVisible({ timeout: 10_000 });

      // Verify no 500 error in network (check response code via API)
      const apiResp = await page.request.post('/api/ingestion/upload', {
        multipart: {
          file: { name: 'test.exe', mimeType: 'application/octet-stream', buffer: Buffer.from('MZ') }
        },
        headers: { 'Authorization': `Bearer ${await getToken(page)}` }
      });
      expect(apiResp.status()).toBe(400);
      const body = await apiResp.json();
      expect(body.code).toBe('unsupported_file_type');
    });
    ```
  </action>
  <acceptance_criteria>
    - Expired token → user redirected to /login within 10s
    - `.exe` upload → HTTP 400 with `code: "unsupported_file_type"` (not 500)
    - Frontend shows user-friendly error message on bad file upload
  </acceptance_criteria>
</task>

<task id="23-04-02" title="LLM timeout + network offline error paths">
  <objective>
    Test that the frontend shows a timeout error when the LLM stub delays > threshold,
    and that the offline banner appears when network is cut.
  </objective>
  <read_first>
    - senso/e2e/real-stack/error-paths.spec.ts — from 23-04-01
    - api/tests/llm_stub_server.py — add delay endpoint
    - senso/src/features/coaching/ChatScreen.tsx — timeout handling, loading state
    - senso/src/components/OfflineBanner.tsx (if exists)
  </read_first>
  <action>
    1. Add a slow-response endpoint to `llm_stub_server.py`:
       ```python
       @app.post("/v1beta/models/slow-{model}:generateContent")
       async def slow_response(model: str):
           await asyncio.sleep(60)  # simulate timeout
           return FIXTURES["default"]
       ```
       OR add a `?delay=60` query parameter to the main endpoint.

    2. Configure the test stack to use the slow stub for LLM timeout tests:
       Set a low timeout (e.g., 8s) via env var `LLM_TIMEOUT_SECONDS=8` in the test
       API service.

    3. Add to `error-paths.spec.ts`:
       ```ts
       test('LLM timeout shows error, not spinner forever', async ({ page, account }) => {
         await loginAs(page, account);
         await page.goto('/chat');

         // Trigger a request that will timeout (use slow stub path)
         // This requires the API to use slow stub URL for this test — configure via env
         await page.getByRole('textbox').fill('__SLOW_RESPONSE_TEST__');
         await page.getByRole('button', { name: /invia/i }).click();

         // Should show error within timeout + buffer (not spinner forever)
         await expect(page.getByText(/errore|timeout|riprova/i))
           .toBeVisible({ timeout: 20_000 });

         // Must NOT still show the loading spinner after timeout
         await expect(page.getByTestId('loading-spinner')).not.toBeVisible();
       });

       test('offline banner appears when network cut', async ({ page, account, context }) => {
         await loginAs(page, account);
         await page.goto('/chat');

         await context.setOffline(true);
         // Trigger a network request
         await page.getByRole('textbox').fill('Test offline');
         await page.getByRole('button', { name: /invia/i }).click();

         // Offline indicator appears
         await expect(page.getByTestId('offline-banner')).toBeVisible({ timeout: 5_000 });

         await context.setOffline(false);
         // Banner disappears when back online
         await expect(page.getByTestId('offline-banner')).not.toBeVisible({ timeout: 10_000 });
       });
       ```
  </action>
  <acceptance_criteria>
    - LLM timeout → error message visible within 20s, no infinite spinner
    - Network offline → offline banner visible
    - Network restored → offline banner hidden
    - All error-paths tests pass in real stack project
  </acceptance_criteria>
</task>
