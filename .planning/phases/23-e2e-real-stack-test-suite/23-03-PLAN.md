# Plan 23-03 — Coach Tool Call E2E (Real Stack)

## Objective

Verify that when the LLM stub returns tool call responses, the tool executor fetches
real data from the DB and returns it in the final response. Asserts resource_card URLs
are from the real catalog (not LLM-invented).

  <read_first>

    Add trigger-based tool call responses to `llm_stub_server.py`.

    The stub must return the same format the backend LLM client expects for a tool call.
    Read `service.py` to find the exact format — it's likely either a Gemini
    `functionCall` part or an OpenAI `tool_calls` array.

    Example Gemini format:
    ```python
    TOOL_CALL_FIXTURES = {
        "profile_query": {
            "candidates": [{
                "content": {
                    "parts": [{"functionCall": {
                        "name": "get_user_profile",
                        "args": {}
                    }}]
                },
                "finishReason": "STOP"
            }]
        },
        "italy_query": {
            "candidates": [{
                "content": {
                    "parts": [{"functionCall": {
                        "name": "search_italy_rules",
                        "args": {"topic": "IRPEF", "locale": "it"}
                    }}]
                },
                "finishReason": "STOP"
            }]
        }
    }
    ```

    The stub should handle the multi-turn conversation: first response is a tool call,
    second request (with tool result) returns the final text response.

    Add triggers: `"quanto ho speso"` → `search_user_transactions` tool call,
    `"mio profilo"` → `get_user_profile` tool call.
  </action>
  <acceptance_criteria>
    - POST to stub with "IRPEF" in message → returns functionCall for search_italy_rules
    - POST with tool result in history → returns final text response
    - Stub handles 2-turn conversation (tool call + tool result → final response)
  </acceptance_criteria>
</task>

<task id="23-03-02" title="Tool call E2E assertions + resource_card URL validation">
  <objective>
    Write E2E tests that verify: tool calls execute against real DB data, response
    contains real numbers from the DB, and resource_card URLs are from the real catalog.
  </objective>
  <read_first>
    - senso/e2e/real-stack/fixtures.ts — account + authedPage fixture
    - api/app/content/ — content catalog for URL verification
  </read_first>
  <action>
    Create `senso/e2e/real-stack/tool-calls.spec.ts`:

    ```ts
    import { test, expect } from './fixtures';

    test('get_user_profile tool call returns real DB data', async ({ page, account }) => {
      // Login first (reuse from fixtures or inline)
      await loginAs(page, account);

      // Ask a question that triggers get_user_profile tool
      await page.goto('/chat');
      await page.getByRole('textbox').fill('Mostrami il mio profilo finanziario');
      await page.getByRole('button', { name: /invia/i }).click();

      const bubble = page.getByTestId('assistant-bubble').last();
      await expect(bubble).toBeVisible({ timeout: 20_000 });

      // Response should mention real user data (not mock data)
      // The tool executor fetched real profile → real numbers in message
      const text = await bubble.textContent();
      expect(text).toBeTruthy();
      expect(text!.length).toBeGreaterThan(20);
    });

    test('search_italy_rules tool call returns real catalog entry', async ({ page, account }) => {
      await loginAs(page, account);
      await page.goto('/chat');
      await page.getByRole('textbox').fill("Spiegami l'IRPEF 2025");
      await page.getByRole('button', { name: /invia/i }).click();

      const bubble = page.getByTestId('assistant-bubble').last();
      await expect(bubble).toBeVisible({ timeout: 20_000 });
      const text = await bubble.textContent();
      // Italy rules BM25 should have returned IRPEF entry
      expect(text).toMatch(/IRPEF|scaglion|23%|28\.000/i);
    });

    test('resource_card URLs are from real catalog (not LLM-invented)', async ({ page, account }) => {
      await loginAs(page, account);
      await page.goto('/chat');
      await page.getByRole('textbox').fill('Posso comprare un iPhone da 1200 euro?');
      await page.getByRole('button', { name: /invia/i }).click();

      const cards = page.getByTestId('resource-card');
      const count = await cards.count();
      for (let i = 0; i < count; i++) {
        const link = await cards.nth(i).getByRole('link').getAttribute('href');
        // URL must not be null and must be a real URL (not invented)
        expect(link).toBeTruthy();
        expect(link).toMatch(/^https?:\/\//);
        // Must not contain LLM hallucination patterns
        expect(link).not.toContain('example.com');
        expect(link).not.toContain('undefined');
      }
    });
    ```
  </action>
  <acceptance_criteria>
    - Tool call tests pass against real stack (not mocked)
    - Profile tool returns non-empty response with real user context
    - Italy rules tool returns response mentioning IRPEF numbers
    - Resource card URLs are real (not null, not example.com, not LLM-invented)
    - All tests pass in `npx playwright test --project=real-stack tool-calls.spec.ts`
  </acceptance_criteria>
</task>
