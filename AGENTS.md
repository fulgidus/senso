## Project

**S.E.N.S.O. - Sistema Educativo per Numeri, Spese e Obiettivi**

Modular AI financial education platform. Users upload financial documents, get structured profiles, and receive personalized coaching via voice or text. Built for extensibility — the core ingestion, profiling, and coaching pipelines are provider-agnostic and domain-adaptable.

**Core Value:** Help users make better financial decisions by combining their real financial data with AI guidance and actionable education.

### Constraints

- **Audience**: Primary target is young adults with low financial literacy — language and UX must be simple, direct, and jargon-light.
- **AI Centrality**: AI drives the core experience (document ingestion, profile reasoning, conversational guidance) — not decorative.
- **Data/Safety**: Financial guidance must avoid unsafe advice patterns and enforce persona boundaries consistently.
- **i18n**: Italian-first (`it` locale is primary). Every user-facing string, content item, and LLM-generated card must carry a `locale` field. The coaching LLM replies in the locale of the request. Content search filters by locale. Never hardcode language strings in source code.
- **Modularity**: Ingestion modules, LLM providers, content catalogs, and coaching personas are pluggable. New document types, languages, or AI providers should require config changes, not architectural rewrites.

## Conventions

### Running Services / Tests

**Always use Docker Compose to run the stack.** Never invoke `uv run`, `python`, or `pnpm` directly on the host for server startup, test execution, or build verification.

```bash
docker compose run --rm api uv run pytest
docker compose run --rm frontend pnpm build
docker compose run --rm api /app/.venv/bin/python -c "import app; print('ok')"
```

### i18n Rules

- Italian (`it`) is the primary locale. English (`en`) mirrors it.
- Every content item **must carry a `locale` field**.
- The coaching LLM replies in the locale of the request.
- Content search **must filter by locale** before prompt injection.
- Never hardcode language strings in source code — use `fe/src/i18n/locales/{it,en}.json`.
- i18n keys use dot notation: `coaching.verdictYes`, `content.readingMinutes`, etc.

### LLM and Prompt Conventions

- All LLM calls go through `llm_client.complete(route=...)` or `llm_client.vision(route=...)` — never call provider SDKs directly.
- Prompt templates use `.j2` files via `jinja2.Environment` + `FileSystemLoader` — never inline multiline strings.
- Schema contracts use `.schema.json` files. Conformance enforced via `response_format` structured output, NOT prose. `jsonschema.validate()` is post-hoc safety net only.
- `additionalProperties: false` in all structured output schemas.
- `loader.py` exposes `get_schema(name)` — use it.

### Database Conventions

- No Alembic. Schema changes via `_add_missing_columns()` in `session.py`.
- **No mixed ORM + raw connections.** Migrations use raw SQL. App CRUD uses ORM sessions. Never hold both open on the same table.
- `user_profiles.insight_cards` — generated during document ingestion.
- `user_profiles.coaching_insights` — generated by coach at chat time (appended, not overwritten).
- No vector store. Content search uses BM25 in-process exposed as LLM tool call.

### UI Error State Convention

**A UI component that fails to load data must show an error state, not an empty set.** Skeleton placeholders are only for the *loading* phase. Once the fetch completes:

- **Success with data** → render content.
- **Success with empty result** → explicit "no data yet" message with guidance.
- **Fetch error** → error indicator with retry action.

Never conflate "still loading", "loaded but empty", and "failed" into the same visual.

### Content Catalog Conventions

- Static JSON files at `be/app/content/`.
- Each item: `id`, `locale`, `type`, `title`, `topics: string[]`, plus type-specific fields.
- BM25 index built at startup, locale-aware.
- LLM receives search results (id, type, title, summary, score) via tool call, never full catalog.

### Card and Component Conventions

- `action_type: "calculator"` → interactive miniapp (e.g. loan calculator).
- `action_type: "funnel"` → partner offer card. Catalog at `be/app/content/partners.json`.
- `affordability_verdict`: `"yes"` | `"no"` | `"conditional"` | `null`.
- A2UI `details_a2ui` — supplementary number comparison panels. Null for qualitative answers.
- `resource_cards.url` must be a real URL from the catalog — never LLM-invented.
